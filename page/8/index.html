leetCode-1262-可被三整除的最大和

发表于 2020-04-02 分类于 数据结构与算法 ， leetcode
本文字数： 2.2k 阅读时长 ≈ 2 分钟

题目描述

给你一个整数数组 nums，请你找出并返回能被三整除的元素最大和。

示例 1：

输入：nums = [3,6,5,1,8]

输出：18

解释：选出数字 3, 6, 1 和 8，它们的和是 18（可被 3 整除的最大和）。

示例 2：

输入：nums = [4]

输出：0

解释：4 不能被 3 整除，所以无法选出数字，返回 0。

示例 3：

输入：nums = [1,2,3,4,4]

输出：12

解释：选出数字 1, 3, 4 以及 4，它们的和是 12（可被 3 整除的最大和）。

解题思路

这个题目的主要思路就是分析3的特点。比如数组[4,5]返回能被三整除的元素最大和，4%3得到1，5%3得到2，那么这两个元素取模的和为三，所以加起来也是3的倍数。

用一个二维数组做dp运算。第一维数组是遍历到第n个元素时n和n前面被三整除的元素最大和。因为在求被三整除的元素最大和时，需要记录前面取模的结果(因为如果当前元素取模+前面取模结果 = 3 说明结果中可以加上这个结果)所以dp状态的定义有三种：

  • 当前状态模3为0的最大和
  • 当前状态模3为1的最大和
  • 当前状态模3为2的最大和

当前状态模3为1，当前状态模3为2都是辅助状态用于求出模3为0的状态。所以动态转移方程中是根据当前数字模3的结果和前面状态的结果相加是否为3得到的。状态转移方程为:dp[i][(0-2)] = max{dp[i-1][(0-2)],dp[i-1][(0-2)] + nums[i]}

leetCode-1248-统计「优美子数组」

发表于 2020-03-29 更新于 2019-03-29 分类于 数据结构与算法 ， leetcode
本文字数： 2.7k 阅读时长 ≈ 2 分钟

题目描述

给你一个整数数组 nums 和一个整数 k。

如果某个 连续 子数组中恰好有 k 个奇数数字，我们就认为这个子数组是「优美子数组」。

请返回这个数组中「优美子数组」的数目。

示例1：

输入：nums = [1,1,2,1,1], k = 3

输出：2

解释：包含 3 个奇数的子数组是 [1,1,2,1] 和 [1,2,1,1] 。

示例 2：

输入：nums = [2,4,6], k = 1

输出：0

解释：数列中不包含任何奇数，所以不存在优美子数组。

示例 3：

输入：nums = [2,2,2,1,2,2,1,2,2,2], k = 2

输出：16

提示：

  • 1 <= nums.length <= 50000

  • 1 <= nums[i] <= 10^5

  • 1 <= k <= nums.length

解题思路

仔细分析题目会发现，构成的优美子数组和数字本身大小没有关系，而是和数字的奇偶性和出现的位置有关系。下面分析一下：

  • 假如nums数组是[1,2,3,4,5,6],k是2
  • 那么可以将数组抽象为一个[奇，偶，奇，偶，奇，偶]的数组
  • 那么构成的优美子数组里面要求只能有两个奇数的数组。
  • 那么下标为[0，1，0]即[1,2,3]组合满足要求，那么再给前面火后面数组加上若干偶数，也是满足要求的。因为[1,2,3]构成优美子数组,[1,2,3,4]也是可以构成优美子数组的。
  • 所以再找数组的时候，我们只需要关心构成的数组左边界和右边界都为奇数，和构成数组的前面有多少个偶数，后面有多少个偶数。

所以遍历数组的时候，记录每个奇数的位置，找到连续奇数个数为k的个数。所以可以新建一个记录数组奇数位的下标，通过遍历新建数组来计算组合的数量。

但是上面那种解法需要开辟一个O(n)的空间。我们可以通过一次遍历，通过记录当前遍历到的奇数的个数count，构成优美子数组的左边界left，右边界right。以及左边界前面偶数的个数pre,后面偶数的个数post方式来实现O(1)空间的开销。

题目描述

由 n 个连接的字符串 s 组成字符串 S，记作 S = [s,n]。例如，[“abc”,3]=“abcabcabc”。

如果我们可以从 s2 中删除某些字符使其变为 s1，则称字符串 s1 可以从字符串 s2 获得。例如，根据定义，”abc” 可以从 “abdbec” 获得，但不能从 “acbbe” 获得。

现在给你两个非空字符串 s1 和 s2（每个最多 100 个字符长）和两个整数 0 ≤ n1 ≤ 106 和 1 ≤ n2 ≤ 106。现在考虑字符串 S1 和 S2，其中 S1=[s1,n1] 、S2=[s2,n2] 。

请你找出一个可以满足使[S2,M] 从 S1 获得的最大整数 M 。

说明：

示例：

输入：

s1 =”acb”,n1 = 4

s2 =”ab”,n2 = 2

返回：
2

解题思路

先说一下题意，假如给定的s1为[“abc”,3],和s2为[“ac”,2],那么要求就是s2中两个“ac”在3个“abc”中出现的次数。

一种最暴力的做法就是把s1和s2展开来求。就是遍历n次s1，找到出现n2次s2的次数。比如上面例子：

  • 3个s1展开为“abcabcabc”。
  • 两个s2展开为“acac”。
  • 可以得出“abcabcabc”中有1个“acac”。

但是直接遍历会超时，哪里可以优化，就是找出循环节。也是题目的难点。上面例子不明显，下面再看一个例子：

  • 比如s1为[“aaa”,n1],s2为[“aa”,n2]。
  • 假如遍历一个s1中的字符串”aaa”,s2中的”aa”可以匹配一次。
  • 假如遍历两个即”aaaaaa”,s2中的”aa”可以匹配三次。
  • 遍历4个和6个，得出的都是遍历连个时结果的倍数。
  • 这时候已经找到规律了。就是找到循环体的所在，用计算来找出答案。

在找循环节时，判断是相当复杂的，上面只举了一种简单的情况。具体下面代码部分解释的比较清楚。

leetCode-11-盛最多水的容器

发表于 2020-03-26 更新于 2019-03-26 分类于 数据结构与算法 ， leetcode
本文字数： 967 阅读时长 ≈ 1 分钟

题目描述

给你 n 个非负整数 a1，a2，…，an，每个数代表坐标中的一个点 (i, ai) 。在坐标内画 n 条垂直线，垂直线 i 的两个端点分别为 (i, ai) 和 (i, 0)。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

说明：
你不能倾斜容器，且 n 的值至少为 2。

示例：

输入：[1,8,6,2,5,4,8,3,7]

输出：49

解题思路

最一开始看到这个题目，想到的是用dp来做，dp状态定义为当前遍历到第i个数时容器能够容纳水的最大值，但是状态转移的方程是找不到的，因为可能再遍历到第n个数时可以代替dp[i-1]的答案太多了，几乎要重新找一遍。所以dp是走不通的。

后面看到这是一个双指针中很经典的问题，有必要好好分析一下。

  • 先比较左指针和右指针的高度，容器盛放水的容量取决于高度较低的一方，假设左指针指向的是高度较低的一方，那么说明了左指针和其他的边所能盛放的最大容量就是和右指针所指向的边构成的容器。
  • 为什么这时候是最大那，当一开始两个指针一个指向开头一个指向结尾，此时容器的底是最大的，接下来随着指针向内移动，会造成容器的底变小，而所能盛水的高度不会超高左指针的高度，所以相当于找到了以左指针为边的最大的容量。
  • 这时候可以挪动高度较小指针到下一个点，已经找到了两个指针中较小为边的最大的容量，所以后面不用再考虑较小边的其他情况了。同理两个指针向中间挪动，每挪动一次，就找到了以较小高度的边和其他边构成的最大容量。

leetCode-556-下一个更大元素III

发表于 2020-03-25 更新于 2019-03-25 分类于 数据结构与算法 ， leetcode
本文字数： 1.5k 阅读时长 ≈ 1 分钟

题目描述

给定一个32位正整数 n，你需要找到最小的32位整数，其与 n 中存在的位数完全相同，并且其值大于n。如果不存在这样的32位整数，则返回-1。

示例 1:

输入: 12

输出: 21

示例 2:

输入: 21

输出: -1

解题思路

题目的意思是在原数字上进行位的调整从而找到一个大于n的最小整数。

原数字n可以调整需要满足的条件是从高位到低位不可以是递减的。比如数字321，无论怎么调整都得不到一个大于321的数字。所以思路就是找到不是递减的逆序对，为了满足最小大于n的数字，所以要从数字的低位开始遍历。这里有一个坑就是当从低位到高位遍历的过程中当找到那个逆序对时，交换并不说明是最小的数字，比如15321,1和5不是降序的，所以直接交换1和5得到51321，但是答案是21135。所以还需要再从后面遍历找到第一个大于1的数字交换后的数字和1交换，这里是和2交换，然后再将交换后的数字从小到大排序。

leetCode-55-跳跃游戏

发表于 2020-03-24 更新于 2019-03-24 分类于 数据结构与算法 ， leetcode
本文字数： 977 阅读时长 ≈ 1 分钟

题目描述

给定一个非负整数数组，你最初位于数组的第一个位置。

数组中的每个元素代表你在该位置可以跳跃的最大长度。

判断你是否能够到达最后一个位置。

示例 1：

输入: [2,3,1,1,4]

输出: true

解释: 我们可以先跳 1 步，从位置 0 到达 位置 1, 然后再从位置 1 跳 3 步到达最后一个位置。

示例 2:

输入: [3,2,1,0,4]

输出: false

解释: 无论怎样，你总会到达索引为 3 的位置。但该位置的最大跳跃长度是 0 ， 所以你永远不可能到达最后一个位置。

解题思路

可以用贪心和动态规划来解决。判断能不能到达最后一点，可以用最后一点出发，前一步如果想到达这一点，那么前一步不能为0，如果为0了，这时候只能从倒数第二步跳过来，那么对倒数第二步的要求就是最大跳跃长度必须大于等于2，假如倒数第二步可以到达终点，只需把倒数第二步作为终点，向前推即可。

leetCode-43-字符串相乘

发表于 2020-03-22 更新于 2019-03-22 分类于 数据结构与算法 ， leetcode
本文字数： 3.2k 阅读时长 ≈ 3 分钟

题目描述

给定两个以字符串形式表示的非负整数 num1 和 num2，返回 num1 和 num2 的乘积，它们的乘积也表示为字符串形式。

示例 1:

示例 1:

输入: num1 = “2”, num2 = “3”

输出: “6”

示例 2:

输入: num1 = “123”, num2 = “456”

输出: “56088”

说明：

num1和num2的长度小于110。

num1 和num2 只包含数字0-9。

num1 和num2均不以零开头，除非是数字 0 本身。

不能使用任何标准库的大数类型（比如 BigInteger）或直接将输入转换为整数来处理。

解题思路

题目中要求不能直接乘的方式求两个较大的数字的积。第一个想到的思路就是遍历num1的每一位，和num2的每一位相乘，虽然思路很简单但是代码中的坑还是有的。只是这样做，用到了大量字符串的拼接，在提交的时候，执行用时和内存消耗都很高。

还有一种思路就是 num1 位数为 M，num2 位数为 N， num1 x num2 结果 res 最大总位数为 M+N。相乘的方式和上面不一样，上面是平常计算乘法的方式。这一种是不一样的,比如：

  • 假如num1为”123”，num2为”456”
  • 先求出num1的最后一位3，num2最后一位6的乘积为18
  • 在求出num1的倒数第二位2，num2最后一位6的乘积为12
  • 在求出num1的倒数第三位1，num2最后一位6的乘积为6
  • 控制好移位，通过18 + 120 + 600的方式来计算结果。

这样做的好处就是，可以通过建立以一个大的数组，通过循环的方式避免了大量字符串，毕竟直接操作数组开销是远远小于操作字符串的开销。
