leetCode-136-只出现一次的数字

发表于 2020-05-08 分类于 数据结构与算法 ， leetcode
本文字数： 497 阅读时长 ≈ 1 分钟

题目描述

给定一个非空整数数组，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。

说明：

你的算法应该具有线性时间复杂度。 你可以不使用额外空间来实现吗？

示例1:

输入: [2,2,1]

输出: 1

示例 2:

输入: [4,1,2,1,2]

输出: 4

解题思路

为了满足题目说明里面线性时间复杂度不使用额外空间的要求排序和哈希表解法是不能满足要求的，题目思路就是异或的特点，异或就是二进制不进位的相加，所以每两个相同元素异或都等于他们本身。所以遍历每个元素做异或，最后得到的结果就是出现一次的元素。

leetCode-382-链表随机节点

发表于 2020-05-07 分类于 数据结构与算法 ， leetcode
本文字数： 1.8k 阅读时长 ≈ 2 分钟

题目描述

给定一个单链表，随机选择链表的一个节点，并返回相应的节点值。保证每个节点被选的概率一样。

进阶:

  • 如果链表十分大且长度未知，如何解决这个问题？你能否使用常数级空间复杂度实现？

示例:

初始化一个单链表 [1,2,3].

ListNode head = new ListNode(1);

head.next = new ListNode(2);

head.next.next = new ListNode(3);

Solution solution = new Solution(head);

getRandom()方法应随机返回1,2,3中的一个，保证每个元素被返回的概率相等。

solution.getRandom();

解题思路

假如不考虑进阶用O(n)的做法直接求解的话，在Random函数中根据链表长度随机产生一个随机数，然后遍历链表到随机数的位置就可以了。

在进阶里面，假设链表的长度无限大，这样就不能根据链表的长度再去求随机数了，这里用到了一个蓄水池抽样算法，下面简单了解下原理：

蓄水池抽样算法是在面对一些特别大的数据，不能一次计算出长度或者一次遍历完的数据去随机抽取数据的算法，既然是随机抽取那么每个数据的被抽中的概率都是一样的，如何保证每个数据被选择的概率是相同的是随机抽样的关键。假设一个超级大的数据总量为N，想要从总量N里随机抽取M个样品，这时候先把N的前M个数据保存起来，从N+1个数据开始遍历：

  • 当遍历到第N+1个数据时，此时总共遍历的数据是N+1个，但是可以保存的数据是N个，所以每个数据被保存起来的概率是N/N+1,所以当遍历到第N+i个数据时，每个数据被保存起来的概率是N/N+i，怎么去确定保存的数据是多少那？在每遍历一个数据，都对当前遍历到所有的总数(N+i)取这个区间的一个随机数，因为这个随机数的概率是1/N+i，再用随机数对N取模，如果结果在N的范围内，就将数字替换掉，如果不在N的范围内，就不替换，相当于用N/N+i的概率选取出原本保存的数据这样就保证了原本被保存起来数据的概率有N/N+i被取出，同样保证了遍历到的数据有了N/N+i的概率保存起来。这样做的好处就是保证了N+i个数据保存N个数据的随机性，确定了每个数据被保存的概率都是N/N+i。

leetCode-355-设计推特

发表于 2020-05-05 分类于 数据结构与算法 ， leetcode
本文字数： 5.7k 阅读时长 ≈ 5 分钟

题目描述

设计一个简化版的推特(Twitter)，可以让用户实现发送推文，关注/取消关注其他用户，能够看见关注人（包括自己）的最近十条推文。你的设计需要支持以下的几个功能：

  • postTweet(userId, tweetId): 创建一条新的推文
  • getNewsFeed(userId):
  • 检索最近的十条推文。每个推文都必须是由此用户关注的人或者是用户自己发出的。推文必须按照时间顺序由最近的开始排序。
  • follow(followerId, followeeId): 关注一个用户
  • unfollow(followerId, followeeId): 取消关注一个用户

示例 1：

Twitter twitter = new Twitter();

// 用户1发送了一条新推文 (用户id = 1, 推文id = 5).

twitter.postTweet(1, 5);

// 用户1的获取推文应当返回一个列表，其中包含一个id为5的推文.

twitter.getNewsFeed(1);

// 用户1关注了用户2.

twitter.follow(1, 2);

// 用户2发送了一个新推文 (推文id = 6).

twitter.postTweet(2, 6);

// 用户1的获取推文应当返回一个列表，其中包含两个推文，id分别为 -> [6, 5].

// 推文id6应当在推文id5之前，因为它是在5之后发送的.
twitter.getNewsFeed(1);

// 用户1取消关注了用户2.

twitter.unfollow(1, 2);

// 用户1的获取推文应当返回一个列表，其中包含一个id为5的推文.

// 因为用户1已经不再关注用户2.

twitter.getNewsFeed(1);

解题思路

题目中要求设计一个简单的推特，主要功能是关注和取关一个人，创建一个推文，检索最近十条推文。下面一个功能一个功能的来分析。

  • 关注一个人：当关注一个人，需要根据当前的用户的id检索到关注用户的id，因为关注的对象可以是多个并且不能是重复，所以把关注的对象保存到一个set集合中，并且把本用户的id和set集合建立映射，这样就可以根据用户id找到对应的集合了。
  • 取消关注一个人：取消关注一个人可以直接从集合中删除掉。
  • 创建一个推文和获取推文列表是最难的地方(获取推文的要求是自己和关注者的最近10条动态)
    • 需要记录多个用户推文发送的顺序，那么肯定要定义一个时间戳，在每个用户发动态的时候，都要记录当前时间，保存这个时间戳。
    • 发动态需要保存两个状态，第一个状态是发送推文的内容，第二个是发送推文的时间。怎么储存这两个数据，可以重新定义一个推文对象来保存推文的发送时间和内容。一个用户可能发送多个推文，所以需要把一个用户发送的所有的推文连接到一个链表里面，再把链表和用户的id建立映射，这样就可以保证根据用户id来获取这个用户发送的所有推文了。
    • 在获取推文的时候，需要把自己关注的用户的文章也展示出来(最近10条)，并且按照时间排序，所以问题就相当于在多个链表中，根据时间戳排序的最后10个结果。这个问题就类似于leetcode23题合并k个有序链表，不同的是这个只需要取出前十条的结果，在解决合并k个有序链表时，主要有类似于快排的分治和小顶堆两种解法，这个问题中用优先队列解决。
    • 这里大概就是先把自己和关注用户的推文链表添加到队列中，队列一次返回所有链表节点中最近时间发送的推文。具体的多个排序链表的合并可以参考leetcode23还是比较费脑筋的。

题目描述

在一个由 0 和 1 组成的二维矩阵内，找到只包含 1 的最大正方形，并返回其面积。

示例:

输入:

输出: 4

解题思路

这个题目的思路要向在矩阵中正方形特点，矩阵中正方形特点是本身为1，并且右边点也为1，下边也为1，下方也为1，右下也为1，这样就构成了边长为2的正方形，同理可推出边长更长的边。顺着这个思路，写了最暴力的写法，就是遍历每个点查看每个点的能向右下延深多少。但是这样做有大量重复计算，直接超时了。可以动态规划来记录每个计算过的边，dp[i][j]来表示以[i][j]右下角，且只包含1的正方形的边长最大值。

  • 如果该位置的值是 0，则说明这个点结尾的不能构成正方形 dp[i, j] = dp[i,j]。
  • 如果该位置的值是 1，则 dp[i, j] 的值由其上方、左方和左上方的三个相邻位置决定的。具体而言，当前位置的元素值等于三个相邻位置的元素中的最小值加 1，状态转移方程如下：
    dp[i, j]=min(dp[i−1, j], dp[i−1, j−1], dp[i, j−1])+1。

给定一个非负整数数组，你最初位于数组的第一个位置。

数组中的每个元素代表你在该位置可以跳跃的最大长度。

你的目标是使用最少的跳跃次数到达数组的最后一个位置。

示例:

输入: [2,3,1,1,4]

输出: 2

解释: 跳到最后一个位置的最小跳跃数是 2。
从下标为 0 跳到下标为 1 的位置，跳 1 步，然后跳 3 步到达数组的最后一个位置。

leetCode-10-正则表达式匹配

发表于 2020-04-25 分类于 数据结构与算法 ， leetcode
本文字数： 3.9k 阅读时长 ≈ 4 分钟

题目描述

给你一个字符串s和一个字符规律p，请你来实现一个支持 '.'和'*'的正则表达式匹配。

'.' 匹配任意单个字符
'*' 匹配零个或多个前面的那一个元素
所谓匹配，是要涵盖整个字符串s的，而不是部分字符串。

说明:

  • s可能为空，且只包含从a-z的小写字母。
  • p可能为空，且只包含从a-z的小写字母，以及字符.和*。

示例 1:

输入:

s = “aa”

p = “a”

输出: false

解释: “a” 无法匹配 “aa” 整个字符串。

示例 2:

输入:

s = “aa”

p = “a*”

输出: true

解释: 因为 ‘*’ 代表可以匹配零个或多个前面的那一个元素, 在这里前面的元素就是 ‘a’。因此，字符串 “aa” 可被视为 ‘a’ 重复了一次。

示例 3:

输入:

s = “ab”

p = “.*”

输出: true

解释: “.*” 表示可匹配零个或多个（’*’）任意字符（’.’）。

示例 4:

输入:

s = “aab”

p = “c*a *b”

输出: true

解释: 因为 ‘*’ 表示零个或多个，这里 ‘c’ 为 0 个, ‘a’ 被重复一次。因此可以匹配字符串 “aab”。

解题思路

剑指offer原题。

第一种思路是回溯的思路，下面分析一下：

  • 为了方便分析，先不讨论*的情况，那么直接挨个遍历字符串s和字符串p，要么s和p的字符相等，要么p的.可以和s中的字符匹配。
  • 这时候讨论*号，分多少种情况那，*位置一定出现在某个字符的后面或者出现在.的后面。这时候可以分几种情况去讨论。
    • 假如s中的第i个字符和p中的第j个字符不匹配，那么还需要再去遍历p后面一个字符，比如s = 'bb' ,p = a*bb,既是遍历到的元素不相等，如果p后面字符是*。也可以说明匹配是成功的。
    • 假如p的第i个元素在s中出现多次，例如s =aaabb,p = a*bb,这时候不去移动p的指针，而是一直移动s的指针看看是否和a相同。

第二种思路是动态规划：

  • 说到动态规划，第一步先找到dp的状态。二维数组的dp[i][j] 表示 s 的前 i 个是否能被 p 的前 j个匹配。重点在于动态规划的状态转移方程怎么求。这也需要分情况讨论。

    • 假如求出了dp[i - 1][j - 1],那么假如第i和元素和第j个元素相等的话，那么dp[i][j] = dp[i - 1][j - 1]。
    • 同理假如p[j]个元素为.也可以说明匹配上了，即dp[i][j] = dp[i - 1][j - 1]。

  • 分析完普通情况了，怎么处理p[j] = * 这种情况那？

    • 首先给了*，明白 *的含义是 匹配零个或多个前面的那一个元素，所以要考虑他前面的元素 p[j-1],* 跟着他前一个字符走，前一个能匹配上 s[i]，* 才能有用，前一个都不能匹配上 s[i]，* 也无能为力，只能让前一个字符消失，也就是匹配0次前一个字符。

    • 所以按照 p[j-1] 和 s[i] 是否相等，我们分为两种情况：

    • 假如p[j-1]不等同于s[i],这时候就是前一个字符匹配不上的情况，相当于p[j - 1]个字符出现0次,所以这种情况的状态转移方程为dp[i][j] = dp[i][j - 2]。

    • 假如p[j-1]等同于s[i]，相当于p[j - 1] = '.'或者p[j - 1] = s[i]两种情况，这时候状态可能是前面多个状态转移过来的。

      • 例如aaaa和aaab*假如s和p前三个字符都匹配上了，这时候i = 4, j = 4,这时候就算aaa后面加上若干b都是不影响结果的,所以这种情况只与i的前一个状态有关，即dp[i][j] = dp[i-1][j]
      • 例如aabb和aabb*这时候p中的*可以是不存在的时候，即dp[i][j] = dp[i][j - 1]。
      • 例如aaab和aaabb*,这一种情况是去掉多余的 b *，这时候状态和p里面j-2元素有关，即dp[i][j] = dp[i][j - 2]
      • 总之，以上三种情况有一种为真的话都可以说明当前字符是可以匹配上的。

      上面情况总结一下：

  • 如果 p.charAt(j) == s.charAt(i) : dp[i][j] = dp[i-1][j-1]

  • 如果 p.charAt(j) == '.' : dp[i][j] = dp[i-1][j-1]

  • 如果 p.charAt(j) == '*'：

    • 如果 p.charAt(j-1) != s.charAt(i) : dp[i][j] = dp[i][j-2]
    • 如果 p.charAt(i-1) == s.charAt(i) or p.charAt(i-1) == '.'：
      • dp[i][j] = dp[i-1][j]
      • dp[i][j] = dp[i][j-1]
      • dp[i][j] = dp[i][j-2]

leetCode-53-最大子序和

题目描述

给定一个整数数组 nums ，找到一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

示例:

输入: [-2,1,-3,4,-1,2,1,-5,4],

输出: 6

解释: 连续子数组 [4,-1,2,1] 的和最大，为 6。

进阶:

如果你已经实现复杂度为 O(n) 的解法，尝试使用更为精妙的分治法求解。

解题思路

直接用动态规划的思想，dp[i]状态表示以第i个元素结尾时连续子数组的最大和，注意并不是第i个元素时连续最大子序列的和。状态转移的方程主要根据前一个状态是否大于零，如果小于零说明以当前元素结尾的连续序列是他本身，如果大于零则加上之前的状态即可。
