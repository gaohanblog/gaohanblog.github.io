leetCode-56-合并区间

发表于 2020-04-23 分类于 数据结构与算法 ， leetcode
本文字数： 2.3k 阅读时长 ≈ 2 分钟

题目描述

给出一个区间的集合，请合并所有重叠的区间。

示例 1:

输入: [[1,3],[2,6],[8,10],[15,18]]

输出: [[1,6],[8,10],[15,18]]

解释: 区间 [1,3] 和 [2,6] 重叠, 将它们合并为 [1,6].

示例 2:

输入: [[1,4],[4,5]]

输出: [[1,5]]

解释: 区间 [1,4] 和 [4,5] 可被视为重叠区间。

解题思路

怎么找到重叠区间，假如两个区间有重叠部分，比如[a,b]和[c,d]两个区间。无外乎两种情况

  • a>=c并且a<=d，这样说明一个集合是另一个集合的子区间。
  • c <= b这样说明了[c,d]和[a,b]有交集。
    两种情况的特点是，两个区间起点和终点(a,b,c,d)必定不是递增的序列。所以在合并的时候，需要找到不是递增序列的所有序列，然后找到所有序列中左侧最小的一个元素，和右侧最大的元素。

为了能顺利找到所有不是递增的序列，将所有序列的起点排序，这样
保证了，最小起点在最左侧，然后在从头遍历数组挨个比较结束区间是不是大于等于下一个区间的起点，如果大于说明存在两个区间存在重叠部分，将两个区间合并。

leetCode-3-无重复字符的最长子串

发表于 2020-04-22 分类于 数据结构与算法 ， leetcode
本文字数： 1.3k 阅读时长 ≈ 1 分钟

题目描述

给定一个字符串，请你找出其中不含有重复字符的 最长子串 的长度。

示例 1:

输入: “abcabcbb”

输出: 3

解释: 因为无重复字符的最长子串是 “abc”，所以其长度为 3。

示例 2:

输入: “bbbbb”

输出: 1

解释: 因为无重复字符的最长子串是 “b”，所以其长度为 1。

示例 3:

输入: “pwwkew”

输出: 3

解释: 因为无重复字符的最长子串是 “wke”，所以其长度为 3。
请注意，你的答案必须是 子串 的长度，”pwke” 是一个子序列，不是子串。

解题思路

滑动窗口的经典例题，主要思路就是维护左右两个指针形成的区间，保证区间内元素都是唯一的。在遍历过程中如果区间元素唯一，挪动右指针，再去判断新增元素是否和前面元素冲突，不唯一则更新左指针到第一次出现重复元素的地方，然后同时将左右指针向后挪动，再去判断区间的有效性。

leetCode-785-判断二分图

发表于 2020-04-21 分类于 数据结构与算法 ， leetcode
本文字数： 2.7k 阅读时长 ≈ 2 分钟

题目描述

给定一个无向图graph，当这个图为二分图时返回true。

如果我们能将一个图的节点集合分割成两个独立的子集A和B，并使图中的每一条边的两个节点一个来自A集合，一个来自B集合，我们就将这个图称为二分图。

graph将会以邻接表方式给出，graph[i]表示图中与节点i相连的所有节点。每个节点都是一个在0到graph.length-1之间的整数。这图中没有自环和平行边： graph[i] 中不存在i，并且graph[i]中没有重复的值。

示例 1:

输入: [[1,3], [0,2], [1,3], [0,2]]

输出: true

解释:

无向图如下:

我们可以将节点分成两组: {0, 2} 和 {1, 3}。

示例 2:

输入: [[1,2,3], [0,2], [0,1,3], [0,2]]
输出: false
解释:
无向图如下:

我们不能将节点分割成两个独立的子集。

注意:

  • graph 的长度范围为 [1, 100]。
  • graph[i] 中的元素的范围为 [0, graph.length - 1]。
  • graph[i] 不会包含 i 或者有重复的值。
  • 图是无向的: 如果j 在 graph[i]里边, 那么 i 也会在 graph[j]里边。

解题思路

二分图的概念就是相邻的节点染色不相同，所以可以用贪心的思路来解，遍历每一个节点(因为可能图中的节点是不连通的)，然后给相邻节点染相反的色，如果在整个染色过程中没有出现冲突说明是二分图，如果在遍历过程中发现某个节点的颜色和子节点相同，则说明出现冲突不是二分图。

将两个升序链表合并为一个新的升序链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

示例：

leetCode-202-快乐数

发表于 2020-04-19 分类于 数据结构与算法 ， leetcode
本文字数： 916 阅读时长 ≈ 1 分钟

题目描述

编写一个算法来判断一个数 n 是不是快乐数。

「快乐数」定义为：对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和，然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。如果 可以变为 1，那么这个数就是快乐数。

如果 n 是快乐数就返回 True ；不是，则返回 False 。

示例 1：

输入：19

输出：true

解释：

12 + 92 = 82

82 + 22 = 68

62 + 82 = 100

12 + 02 + 02 = 1

解题思路

这个题目的问题在于如何找到循环点，如果一个数字是不是快乐数的话，那么一定进入到某个无限循环中去，出现这个无限循环中数字的特点就是每个位置上的数字的平方和的结果是在前面出现过的，所以可以把每次结果都保存起来，如果出现和前面重复的数字说明进入无限循环，则说明不是快乐数。

leetCode-30-串联所有单词的子串

发表于 2020-04-18 分类于 数据结构与算法 ， leetcode
本文字数： 2k 阅读时长 ≈ 2 分钟

题目描述

给定一个字符串 s 和一些长度相同的单词 words。找出 s 中恰好可以由 words 中所有单词串联形成的子串的起始位置。

注意子串要与 words 中的单词完全匹配，中间不能有其他字符，但不需要考虑 words 中单词串联的顺序。

示例 1：

输入：

s = “barfoothefoobarman”,

words = [“foo”,”bar”]

输出：[0,9]

解释：
从索引 0 和 9 开始的子串分别是 “barfoo” 和 “foobar” 。
输出的顺序不重要, [9,0] 也是有效答案。

示例 2：

输入：

s = “wordgoodgoodgoodbestword”,

words = [“word”,”good”,”best”,”word”]

输出：[]

解题思路

最一开始的思路截取和words长度相等的字符，然后和words元素一一比较，但是words数组中的字符串可以出现在字符串s的任意位置是不连续的(可以颠倒顺序)。所以比较起来是比较麻烦的，一开始用排序的思路，就是把words数组中的所有字符串组成一个字符串，然后排序，再将截取的字符串排序，比较是否相等。但是这个做法有漏洞，比如s:[ab]和words:[ba]排序后相等，但是结果是不等的。

要充分利用题目中给定的条件，题目中说到words数组中的字符串长度是相等的，这样就可以确定把字符分割为和words数组中长度相同的字符，把每个单词和出现的次数建立映射关系，可以把它们都存到一map中去，去比较截取的字符串存入map的结果和words存入map的结果就可以了。

leetCode-1095-山脉数组中查找目标值

发表于 2020-04-17 分类于 数据结构与算法 ， leetcode
本文字数： 3.3k 阅读时长 ≈ 3 分钟

题目描述

（这是一个 交互式问题）

给你一个 山脉数组mountainArr，请你返回能够使得mountainArr.get(index)等于target最小的下标 index值。

如果不存在这样的下标 index，就请返回-1。

何为山脉数组？如果数组 A 是一个山脉数组的话，那它满足如下条件：

首先，A.length >= 3

其次，在0 < i< A.length - 1条件下，存在 i 使得：

  • A[0] < A[1] < ... A[i-1] < A[i]
  • A[i] > A[i+1] > ... > A[A.length - 1]

你将不能直接访问该山脉数组，必须通过MountainArray接口来获取数据：

  • MountainArray.get(k)- 会返回数组中索引为k 的元素（下标从 0 开始）
  • MountainArray.length()- 会返回该数组的长度

注意：

对MountainArray.get发起超过 100 次调用的提交将被视为错误答案。此外，任何试图规避判题系统的解决方案都将会导致比赛资格被取消。

为了帮助大家更好地理解交互式问题，我们准备了一个样例 “答案”：https://leetcode-cn.com/playground/RKhe3ave ，请注意这 不是一个正确答案。

示例 1：

输入：array = [1,2,3,4,5,3,1], target = 3

输出：2

解释：3 在数组中出现了两次，下标分别为 2 和 5，我们返回最小的下标 2。

示例 2：

输入：array = [0,1,2,4,2,1], target = 3

输出：-1

解释：3 在数组中没有出现，返回 -1。

提示：

  • 3 <= mountain_arr.length() <= 10000
  • 0 <= target <= 10^9
  • 0 <= mountain_arr.get(index) <=10^9

解题思路

题目大概意思就是要求返回一个山脉数组中的第一次出现给定目标值的下标。并且题目会封装一个数组为了限制调用次数，可以通过函数来获取指定下标的元素和数组长度。

在给定的数据中3 <= mountain_arr.length() <= 10000,如果直接遍历显然是不行的(最多发起超过100次调用)，那么还有一种方法就是常用的二分法。

难点也是二分法到底怎么去分，最开始思路就是先找到这个山脉数组的峰值，也就是转折点，然后用普通的二分法去求target的值，但是传统的二分法求峰值是不行的：

  • 假如数组[1,2,7,6,3,2,1],取mid元素为6，这时候和左右边界比较并不能确定峰值是在左半部分还是右半部分，思路也就是在这里卡住了。
  • 如何用二分求峰值，可以缩小一下求值的范围，比如上面数组取mid元素为6和mid+1对比，如果大于mid+1说明左边的数组是升序的转折点并不在左边。如果小于mid+1说明右边的数组是降序的，转折点不在右面。这样就可以用二分的思想找到数组的峰值
  • 找到峰值以后，那么左侧就是升序数组，右侧是降序数组，这样就可以用二分法，在左边和右边找target元素了。

leetCode-78-子集

发表于 2020-04-16 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

给定一组不含重复元素的整数数组 nums，返回该数组所有可能的子集（幂集）。

说明：解集不能包含重复的子集。

示例:

输入: nums = [1,2,3]

输出:
[
[3],
[1],
[2],
[1,2,3],
[1,3],
[2,3],
[1,2],
[]
]

解题思路

最直接的思路就是直接穷举，遍历n次数组，每遍历一个元素选择添加或者不添加，但是这种算法复杂度是O(n!),但是没办法题目要求返回所有的可能子集，所以必须要穷举每一种可能。

最一开始用dfs回溯的思路，在回溯的过程中要添加或者删除上一次用到的元素，所以感觉用LinkedList性能要好一点，因为LinkedList删除某个元素是O(1)的耗时，而且LinkedList是用双向链表实现的，并且会维护尾节点和头节点，所以查找和删除都是O(1)的。但是在回溯的过程中，删除的都是列表的最后一个元素，所以ArrayList的查找和删除效率也是O(1)的，所以两种做法都耗时都是差不多的。

题目描述

如果一个矩阵的每一方向由左上到右下的对角线上具有相同元素，那么这个矩阵是托普利茨矩阵。

给定一个 M x N 的矩阵，当且仅当它是托普利茨矩阵时返回 True。

示例 1：

输入:
matrix = [
[1,2,3,4],

最直接的思路就是直接遍历左边一列和上面一列。依次向右下角走，判断是否相等。

leetCode-56-数组中数字出现的次数-I

发表于 2020-04-14 分类于 数据结构与算法 ， leetcode
本文字数： 993 阅读时长 ≈ 1 分钟

题目描述

一个整型数组 nums 里除两个数字之外，其他数字都出现了两次。请写程序找出这两个只出现一次的数字。要求时间复杂度是O(n)，空间复杂度是O(1)。

示例 1：

输入：nums = [4,1,4,6]

输出：[1,6] 或 [6,1]

示例 2：

输入：nums = [1,2,10,4,1,4,3,3]

输出：[2,10] 或 [10,2]

限制：

2 <= nums <= 10000

解题思路

这个是位运算的题目，先看下异或的特点，异或两个二进制位数字，相同为0，不同为1，所以一个数字异或本身结果为0。而且异或满足乘法的分配律。所以将原数组所有数字进行异或得到的结果就是那两个出现一次的数字异或的结果。

求出两个数字异或结果，因为两个数字不相同，那么他们结果中必定存在一位为1的位，根据负数在计算机中的存储形式，可以通过&这个数字的负数，得到的就是最低位的1。

得到这个最低位的1之后，再去遍历数组中的数字，找到所有对应位上为1的数字，将这些数字进行异或操作消除重复的数字，就可以得到数组中两个里面出现一次的数字的其中一个数字。
