leetCode-1160-拼写单词

发表于 2020-01-14 分类于 数据结构与算法 ， leetcode
本文字数： 1.9k 阅读时长 ≈ 2 分钟

题目描述

给你一份『词汇表』（字符串数组）words和一张『字母表』（字符串）chars。

假如你可以用chars中的『字母』（字符）拼写出 words中的某个『单词』（字符串），那么我们就认为你掌握了这个单词。

注意：每次拼写时，chars 中的每个字母都只能用一次。

返回词汇表words中你掌握的所有单词的 长度之和。

示例 1：

输入：words = [“cat”,”bt”,”hat”,”tree”], chars = “atach”

输出：6

解释：

可以形成字符串 “cat” 和 “hat”，所以答案是 3 + 3 = 6。

示例 2：

输入：words = [“hello”,”world”,”leetcode”], chars = “welldonehoneyr”
输出：10

解释：

可以形成字符串 “hello” 和 “world”，所以答案是 5 + 5 = 10。

解题思路

这个题目思路就是遍历words的每个单词，计算每个单词的个数，然后把chars字符串字符存到map中。去map中查每个单词是否小于等于map中对应单词的个数，把满足的计数返回即可。

优化：但是上面方式map开销比较大一些，chars字符中总共26个不同字母组成，可以直接用一个26个空间大小的数组来表示每个字符，其中用字符的ascii码表示表示数组下标，

  • a-z 的ascii码是97-122,,每个字符都减去a作为数组的下标
  • 对应字符的个数作为数组下标的值。

map底层也是用数组实现的，所以这样效率更高一点。

leetCode-1143-最长公共子序列

发表于 2020-01-12 分类于 数据结构与算法 ， leetcode
本文字数： 1.8k 阅读时长 ≈ 2 分钟

题目描述

给定两个字符串 text1 和 text2，返回这两个字符串的最长公共子序列。

一个字符串的 子序列 是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。
例如，”ace” 是 “abcde” 的子序列，但 “aec” 不是 “abcde” 的子序列。两个字符串的「公共子序列」是这两个字符串所共同拥有的子序列。

若这两个字符串没有公共子序列，则返回 0。

示例 1:

输入：text1 = “abcde”, text2 = “ace”

输出：3

解释：最长公共子序列是 “ace”，它的长度为 3。

示例 2:

输入：text1 = “abc”, text2 = “abc”

输出：3

解释：最长公共子序列是 “abc”，它的长度为 3。

示例 3:

输入：text1 = “abc”, text2 = “def”

输出：0

解释：两个字符串没有公共子序列，返回 0。

提示:

1 <= text1.length <= 1000
1 <= text2.length <= 1000
输入的字符串只含有小写英文字符。

解题思路

这是一个典型的动态规划的题目，为啥这个问题就是动态规划来解决呢？因为子序列类型的问题，穷举出所有可能的结果都不容易，而动态规划算法做的就是穷举 + 剪枝，或者记忆化处理。所以可以说只要涉及子序列问题，十有八九都需要动态规划来解决，往这方面考虑就对了。

  • 老规矩先找到dp状态定义，dp[i][j]状态就定义为text1前i个字符中在text2中前j个字符最长公共子序列的个数，

  • 那么当text1[i] == text[j]1时表示公共字符又多了一个，所以
    在上一个状态+1，dp[i][j] = dp[i - 1][j - 1]+1。

  • 当text1[i] != text[j]字符不相等，表示遍历到text的i个字符在txet2中的j个字符没有新增加公共的字符，dp状态又表示遍历到为text1前i个字符中在text2中前j个字符最长公共子序列的个数，所以dp[i][j]就等于前面的状态。

  • 前面的状态可分为两种，一种text1遍历第i个字符text2在第j-1位置的结果，或者text1遍历第个i-1字符text2在第j位置的结果，所以当text1[i] != text[j]时dp[i][j] = max(dp[i-1][j],dp[i][j-1])。

leetCode-1071-字符串的最大公因子

发表于 2020-01-11 分类于 数据结构与算法 ， leetcode
本文字数： 1.3k 阅读时长 ≈ 1 分钟

题目描述

对于字符串 S 和 T，只有在 S = T + … + T（T 与自身连接 1 次或多次）时，我们才认定 “T 能除尽 S”。

返回最长字符串 X，要求满足 X 能除尽 str1 且 X 能除尽 str2。

示例 1：

输入：str1 = “ABCABC”, str2 = “ABC”

输出：”ABC”

示例 2：

输入：str1 = “ABABAB”, str2 = “ABAB”

输出：”AB”

示例 3：

输入：str1 = “LEET”, str2 = “CODE”

输出：””

提示：

1 <= str1.length <= 1000

1 <= str2.length <= 1000

str1[i] 和 str2[i] 为大写英文字母

解题思路

这个题目找出两个字符串的最大公共因子，一开始思路是假如两个字符串存在公因子，那么第一个字符串的长度和第二个字符串的长度一定为公因子的倍数，但是具体怎么去找到这个公因子还是没办法。

gcd算法：欧几里德算法又称辗转相除法，是指用于计算两个正整数a，b的最大公约数。

例如：给定m=544，n=119，
则余数r=m%n=544%119=68;

因r!=0，所以跳过上述步骤2，执行步骤3。；

置m=119，n=68，=>r=m%n=119%68=51;

置m=68，n=51，=>r=m%n=68%51=17；

置m=51，n=17，=>r=m%n=51%17=0

所以mn的最大公约数是17

假如两个找到了两个字符串的最大公约数，就剩下判断两个字符串是不是由同一个公因子构成的。假如是由两个公因子构成的那么第一个串就是m个公因子的重复，str2 是 n 个公因子 的重复，连起来就是 m+n 个 公因子，好像 m+n 个 公因子 跟 n+m 个公因子是一样的,即str1+str2=str2+str1

所以 str1 + str2 == str2 + str1 就意味着有解。

leetCode-1013-将数组分成和相等的三个部分

发表于 2020-01-10 分类于 数据结构与算法 ， leetcode
本文字数： 2.1k 阅读时长 ≈ 2 分钟

题目描述

给你一个整数数组 A，只有可以将其划分为三个和相等的非空部分时才返回 true，否则返回 false。

形式上，如果可以找出索引i+1 < j且满足(A[0] + A[1] + ... + A[i] == A[i+1] + A[i+2] + ... + A[j-1] == A[j] + A[j-1] + ... + A[A.length - 1])就可以将数组三等分。

示例 1：

输出：[0,2,1,-6,6,-7,9,1,2,0,1]

输出：true

解释：0 + 2 + 1 = -6 + 6 - 7 + 9 + 1 = 2 + 0 + 1

示例 2：

输入：[0,2,1,-6,6,7,9,-1,2,0,1]

输出：false

示例 3：

输入：[3,3,6,5,-2,2,5,1,-9,4]

输出：true

解释：3 + 3 = 6 = 5 - 2 + 2 + 5 + 1 - 9 + 4

提示：

3 <= A.length <= 50000
-10^4<= A[i] <= 10^4

解题思路

最一开始没理解题意，以为在数组中找出的元素可以是不连续的，仔细看题意是连续的三个非空数组，意思就是把数组分为三段，和相等即返回true。

三段相等，那么每段数组元素的结果为总和的三分之一。第一种做法遍历整个数组如果遍历完找到三个为数组总和的三分之一的三个部分即可。还有一种做法是双指针，左指针和右指针一块向中间夹着找，这样节约时间，整体和前面思路也是差不多的。但是双指针要注意边界，好几次边界问题没通过。

leetCode-72-编辑距离

发表于 2020-01-09 分类于 数据结构与算法 ， leetcode
本文字数： 2.7k 阅读时长 ≈ 2 分钟

题目描述

给定两个单词 word1 和 word2，计算出将 word1 转换成 word2 所使用的最少操作数 。

你可以对一个单词进行如下三种操作：

插入一个字符
删除一个字符
替换一个字符

示例 1:

输入: word1 = “horse”, word2 = “ros”

输出: 3

解释:

horse -> rorse (将 ‘h’ 替换为 ‘r’)

rorse -> rose (删除 ‘r’)

rose -> ros (删除 ‘e’)

示例 2:

输入: word1 = “intention”, word2 = “execution”

输出: 5

解释:

intention -> inention (删除 ‘t’)

inention -> enention (将 ‘i’ 替换为 ‘e’)

enention -> exention (将 ‘n’ 替换为 ‘x’)

exention -> exection (将 ‘n’ 替换为 ‘c’)

exection -> execution (插入 ‘u’)

解题思路

一开始看这个题目感觉无从下手，思路很难找到。看完一些大神的思路，下面顺着动态规划的思想看下推导的过程：

先找到状态，定义数组元素的含义由于我们的目的求将 word1 转换成 word2 所使用的最少操作数 。那我们就定义 dp[i] [j]的含义为：当字符串 word1 的长度为 i，字符串 word2 的长度为 j 时，将 word1 转化为 word2 所使用的最少操作次数为 dp[i] [j]。

怎么理解上面话，假如word1是一个空串，其转换为另一个字符串所需要的最少操作数显然就是另一个字符串的长度。因此对于i == 0的情况，dp[0, j] = j。对于j == 0的情况，dp[i, 0] = i。

理解了前面状态的定义接着看怎么找状态的转移方程：

如果i和j均不为0且此时两个字符相同，显然dp[i, j] = dp[i - 1, j - 1]。因为此时我们不需要考虑word1[i-1]与word2[j-1]之间的转换，word1中[0, i - 1]范围内的字符串转换成word2中[0, j - 1]范围内的字符串所需要的最少操作步骤和word1中[0, i - 2]范围内的字符串转换成word2中[0, j - 2]范围内的字符串所需要的最少操作步骤相同。

如果i和j均不为0且word1[i] != word2[j]，这又分为三种情况。

  • 如果把字符word1[i] 替换成与 word2[j] 相等，则有 dp[i][j] = dp[i-1][j-1] + 1
  • 如果在字符串word1末尾插入一个与word2[j]相等的字符，则有dp[i][j] = dp[i][j-1] + 1
  • 如果把字符word1[i]删除，则有dp[i][j]=dp[i-1] [j]+1
  • 那么我们应该选择一种操作，使得dp[i][j]的值最小，即有dp[i][j] = min(dp[i-1][j-1]，dp[i][j-1]，dp[[i-1][j]]) + 1

leetCode-940-不同的子序列II

发表于 2020-01-08 分类于 数据结构与算法 ， leetcode
本文字数： 1.9k 阅读时长 ≈ 2 分钟

题目描述

给定一个字符串 S，计算 S 的不同非空子序列的个数。

因为结果可能很大，所以返回答案模 10^9 + 7.

示例 1：

输入：”abc”

输出：7

解释：7 个不同的子序列分别是 “a”, “b”, “c”, “ab”, “ac”, “bc”, 以及 “abc”。

示例 2：

输入：”aba”

输出：6

解释：6 个不同的子序列分别是 “a”, “b”, “ab”, “ba”, “aa” >以及 “aba”。

示例 3：

输入：”aaa”

输出：3

解释：3 个不同的子序列分别是 “a”, “aa” 以及 “aaa”。

提示：

S 只包含小写字母。
1 <= S.length <= 2000

解题思路

刚看到这个题目最直接的思路就是，用一个一维动态规划解决，定义dp[i]的状态是前i个字母全排列非空子序列的个数，比如”abc”,dp[0]表示第一个字母只有"a"一种可能,dp[1]有"a" ，"b"，"ab",dp[2]有"a","b","ab","ac","bc","abc","c'那么定义dp的转移方程就是dp[i] = dp[i-1] * 和当前元素组合 + dp[i-1](原来的组合数) + 1(自身也为一个字符串)即为dp[i] = dp[i-1] * 2 + 1。心想hard难度的题真的那么简单吗？

但是这样是行不通的，没有判重，比如”abcc”，假如字符串中出现了重复的字符，上面的方程就是不成立的，因为有了两次c,怎么解决重复字符，第一个c出现的时候和ab进行了全排列，假如第二次c出现这时候，c字符和前一个c出现前面字符向匹配会和前一个c出现重复，所以需要减去前一个出现重复的可能即可，这时候只需要记录上一个c出现的下标，就可以了。这时候动态转移方程可以用两种形式表示,假如前面一个元素没出现过为dp[i] = dp[i-1] * 2 + 1，出现过为dp[i] = dp[i-1]*2 - dp[上个元素下标]。

题目描述

给定一个由 ‘1’（陆地）和 ‘0’（水）组成的的二维网格，计算岛屿的数量。一个岛被水包围，并且它是通过水平方向或垂直方向上相邻的陆地连接而成的。你可以假设网格的四个边均被水包围。

示例 1:

输入:

11110

11010

11000

00000

输出: 1

示例 2:

输入:

11000

11000

00100

00011

解题思路

题意是找到所有连通的1区域。

最开始的思路就是DFS，先从某个为1的点开始，并且用一个辅助数组来标志这个点为访问过，分别搜索上下左右四个点，同样对上下左右四个点中为1的点为基准继续做搜索。这样就会把所有这个区域联通的1找到。这个题目用BFS也可以做。

老师还讲了一个经典的解法就是并查集：

并查集就是有“合并集合”和“查找集>合中的元素”两种操作的关于数据结构的一种算法。

用集合中的某个元素来代表这个集合，该元素称为集合的代表元。
一个集合内的所有元素组织成以代表元为根的树形结。

假如用数组表示对于每一个元素 >parent[x]指向x在树形结构上的父亲节点。如果x是>>根节点，则令parent[x] = x。
对于查找操作，假设需要确定x所在的的集合，也就是确定集合的代表元。可以沿着parent[x]不断在树形结>构中向上移动，直到到达根节点。

路径压缩: 为了加快查找速度，查找时将x到根节点路径上的所有点的parent设为根节点，该优化方法称为压缩路径。

上面是网上找的一些并查集的资料，怎么理解并查集，假如新建一个长度为n的数组并初始化数组中每个元素值为他自己即是arr[n] = n，这时就表示下标为n的指向值为n的元素即自己指向自己，假如让i指向j只需将下标为i的值为j即arr[i] = j就表示i的父节点是j。

用并查集解这个题，遍历整个数组只要相邻的1就把他们连接起来，最后返回并查集中有多少个根的元素就表示有多少个相连通1的区域。

leetCode-322-零钱兑换

发表于 2020-01-05 分类于 数据结构与算法 ， leetcode
本文字数： 1.2k 阅读时长 ≈ 1 分钟

题目描述

给定不同面额的硬币 coins 和一个总金额 amount。编写一个函数来计算可以凑成总金额所需的最少的硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。

示例 1:

输入: coins = [1, 2, 5], amount = 11

输出: 3

解释: 11 = 5 + 5 + 1

示例 2:

输入: coins = [2], amount = 3

输出: -1

说明:
你可以认为每种硬币的数量是无限的。

解题思路

最开始思路就是贪心，先挑大面值的拿，假如大面值的拿完了，然后再去小面值里面去找，但是这种思路是不对的，比如 1， 5， 11，我们要凑出15的时候， 贪心策略就会出错：

  • 先挑大面值11可以拿，再看5不满足，最后看1，得到15 = 11 * 1 + 1 * 4拿了5次
  • 正确答案是直接拿三张五块的。

考虑动态规划，怎么找到当前状态那，顺着上面思路想。

  • 假如需要拼出15，那么先从1开始尝试，先拿出个1块的那么剩下的14块，先不管，因为14也需要动态求解所以 15 = f(14) + 1

  • 还可以先拿出一张5块的那么那么此时：15 = f(10) + 5

  • 还可以先拿出一张11的：15 = f(4) + 11

  • 找出三个结果里面需要硬币数量最少的一个解。

这时候，定义当前状态dp[i]为当前最优解，那么遍历dp[i-不同价值的金币]找到最少硬币就是当前的解。

leetCode-300-最长上升子序列

发表于 2020-01-04 分类于 数据结构与算法 ， leetcode
本文字数： 1.2k 阅读时长 ≈ 1 分钟

题目描述

给定一个无序的整数数组，找到其中最长上升子序列的长度。

示例:

输入: [10,9,2,5,3,7,101,18]

输出: 4

解释: 最长的上升子序列是 [2,3,7,101]，它的长度是 4。
说明:

可能会有多种最长上升子序列的组合，你只需要输出对应的长度即可。
你算法的时间复杂度应该为 O(n^2) 。

解题思路

虽然暴力求解的方法不能通过，先看下暴力求解的过程，然后再去推导出动态规划的方程。

从第一个元素a开始，把数组剩下的元素，挨个遍历，如果b比当前元素a大，再把遍历b和b后面的元素对比，直到把数组遍历完，返回出来以a开头最长上升子序列。然后再遍历a后面的数字。

但是这样是指数级别的复杂度，仔细分析会发现里面有大量的重复运算，比如上面找到b元素后，我们在第一次遍历数组的时候还需要把b重新查一遍。

顺着这个思路，可不可以把计算过的重复元素先保存起来。dp的思路就是dp[i]表示nums[0]到nums[i]，含nums[i]的最长上升子序列，每个dp[i]都表示到i元素的最大序列，所以每个元素代表的是当前最优，并不是全局最优。

状态转移的方程是根据前面状态得到的，遍历到 nums[i] 时，考虑把索引i之前的所有的数都看一遍，只要当前的数nums[i]大于之前的某个数，那么 nums[i] 就可以接在这个数后面形成一个更长的上升子序列。因此，dp[i] 就等于索引 i 之前小于 nums[i] 的状态最大者 +1。
