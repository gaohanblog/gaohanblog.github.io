leetCode-837-新21点

发表于 2020-06-06 分类于 数据结构与算法 ， leetcode
本文字数： 1.6k 阅读时长 ≈ 1 分钟

题目描述

爱丽丝参与一个大致基于纸牌游戏 “21点” 规则的游戏，描述如下：

爱丽丝以 0 分开始，并在她的得分少于 K 分时抽取数字。 抽取时，她从 [1, W] 的范围中随机获得一个整数作为分数进行累计，其中 W 是整数。 每次抽取都是独立的，其结果具有相同的概率。

当爱丽丝获得不少于 K 分时，她就停止抽取数字。 爱丽丝的分数不超过 N 的概率是多少？

示例 1：

输入：N = 10, K = 1, W = 10

输出：1.00000

说明：爱丽丝得到一张卡，然后停止。

示例 2：

输入：N = 6, K = 1, W = 10

输出：0.60000

说明：爱丽丝得到一张卡，然后停止。
在 W = 10 的 6 种可能下，她的得分不超过 N = 6 分。

示例 3：

输入：N = 21, K = 17, W = 10

输出：0.73278

提示：

  • 0 <= K <= N <= 10000
  • 1 <= W <= 10000
  • 如果答案与正确答案的误差不超过 - 10^-5，则该答案将被视为正确答案通过。
  • 此问题的判断限制时间已经减少。

解题思路

题目的思路是从后往前逆向求，定义dp的状态为当前手上点子数位i时能赢得游戏的胜利的概率。

  • 假如给定数据N = 21, K = 17, W = 10。
  • 因为k = 17，所以当i = 16时还有最后一次抽卡的机会。W的范围是10，所以抽完卡的范围为17~26。17~21可以赢得比赛，所以17~21的概率是1(没有机会再抽卡了)。22~26超过了规定，赢得比赛的概率是零。所以当i = 16时此时赢得比赛的概率是后面所有的情况加起来/W，即0.5。
  • 同样当i = 15时，根据W的范围是10，可能出现的情况是16~25,这时候16~25的概率已经求出来了。所以只需要将所有的情况加起来/W，得到0.55。
  • 所以根据上面可以推测出来dp的状态转移方程就是dp[i] = (dp[i + 1]到dp[i + W]的和) / W

leetCode-238-除自身以外数组的乘积

发表于 2020-06-05 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

给你一个长度为n的整数数组nums，其中n > 1，返回输出数组output，其中 output[i]等于nums中除nums[i]之外其余各元素的乘积。

示例:

输入: [1,2,3,4]

输出: [24,12,8,6]

提示： 题目数据保证数组之中任意元素的全部前缀元素和后缀（甚至是整个数组）的乘积都在 32 位整数范围内。

说明: 请不要使用除法，且在O(n) 时间复杂度内完成此题。

进阶：
你可以在常数空间复杂度内完成这个题目吗？（ 出于对空间复杂度分析的目的，输出数组不被视为额外空间。）

解题思路

如果题目不限制除法，可以直接将所有数字的乘积，除去当前数字就是除自身以外数组的乘积。有一种特别巧妙的解法，就是用前缀积和后缀积。

  • 要求除自身以外数组的乘积，只需求出这个数字之前所有的数字乘积和后面的乘积。
  • 所以用两个数组保存所有元素的前缀积和后缀积，前缀积和后缀积的乘积就是结果。
  • 为了满足题目给定的条件，常数空间完成，可以先把前缀积保存到结果数组里面(结果数组不算额外空间)，然后再计算后缀积的时候直接更新到结果里面。

发表于 2020-06-04 分类于 数据结构与算法 ， leetcode
本文字数： 544 阅读时长 ≈ 1 分钟

题目描述

求 1+2+…+n ，要求不能使用乘除法、for、while、if、else、switch、case等关键字及条件判断语句（A?B:C）。

示例 1：

输入: n = 3

输出: 6

示例 2：

输入: n = 9

输出: 45

限制：

1 <= n <= 10000

解题思路

题目说明里没有限制加减逻辑运算。一般思路就是用递归一直从n + n - 1 + n - 2....到1。但是递归出口是判断最后一个数字是否大于1，题目要求不能用条件判断语句，所以可以用&&逻辑运算特性，把递归方程放到后面，递归出口放到前面，这样如果前面不成立了，后面也不会执行了。

leetCode-395-至少有K个重复字符的最长子串

发表于 2020-06-03 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

找到给定字符串（由小写字符组成）中的最长子串 T ， 要求 T 中的每一字符出现次数都不少于 k 。输出 T 的长度。

示例 1:

输入:

s = “aaabb”, k = 3

输出:

3

最长子串为 “aaa” ，其中 ‘a’ 重复了 3 次。

示例 2:

输入:

s = “ababbc”, k = 2

输出:

5

最长子串为 “ababb” ，其中 ‘a’ 重复了 2 次， ‘b’ 重复了 3 次。

解题思路

对于一个字符串来说，如果要求子串最少出现k次，那么如果某些字母出现的次数小于k,说明整个串不可能满足题目要求。那么就需要递归的判断这个不符合字符的左右两边的字符串是否符合标准。

  • 比如”abacbb”,k是要求是2。
  • 最先判断得到c字符不符合要求，这时候去判断左边区间[aba]和右区间[bb]
  • 左区间中b又不符合要求。所以只有右区间[bb]符合要求。

leetCode-863-二叉树中所有距离为 K 的结点

发表于 2020-06-02 分类于 数据结构与算法 ， leetcode
本文字数： 1.6k 阅读时长 ≈ 1 分钟

题目描述

给定一个二叉树（具有根结点 root）， 一个目标结点 target ，和一个整数值 K 。

返回到目标结点 target 距离为 K 的所有结点的值的列表。 答案可以以任何顺序返回。

示例 1：

输入：root = [3,5,1,6,2,0,8,null,null,7,4], target = 5, K = >2

输出：[7,4,1]

解释：

所求结点为与目标结点（值为 5）距离为 2 的结点，
值分别为 7，4，以及 1

提示：

  • 给定的树是非空的，且最多有 K 个结点。
  • 树上的每个结点都具有唯一的值 0 <= node.val <= 500 。
  • 目标结点 target 是树上的结点。
  • 0 <= K <= 1000.

解题思路

第一种思路是二叉树转化为无向连通图，用二维数组来保存连通图的信息，因为node范围0 <= node.val <= 500，所以在空间上是可以接受的。然后再dfs遍历结点为与目标结点距离为k的结点。

leetCode-84-柱状图中最大的矩形

发表于 2020-05-24 分类于 数据结构与算法 ， leetcode
本文字数： 2.3k 阅读时长 ≈ 2 分钟

题目描述

给定 n 个非负整数，用来表示柱状图中各个柱子的高度。每个柱子彼此相邻，且宽度为 1 。

求在该柱状图中，能够勾勒出来的矩形的最大面积。

示例:

输入: [2,1,5,6,2,3]

输出: 10

解题思路

最直接的思路是穷举，因为柱状图里面的矩形最大都是取决于最小的柱子的高度，所以在穷举的过程中，找到最大的矩形的面积即可。

第二种思路就是单调栈。在穷举的过程中有大量的重复运算，在遍历数组的过程中，可以直接找到已经可以确定的一些柱形的面积。例如

  • 给定数组[2,1,5,6,2,3],遍历到第二个元素1时，这时候就可以确定和第一个高度为2的柱体所能构成的最大面积就是他本身。
  • 假如遍历到第三个元素时柱高为5，这时候不能确定前一个高度（因为前一个高度是1，小于当前高度）。
  • 所以遍历每个柱体可以用栈来保存前面不能确定高度的柱体，若当前的柱体高度大于等于栈顶柱体的高度，就直接将当前柱体入栈，否则若当前的柱体高度小于栈顶柱体的高度，说明当前栈顶柱体找到了右边的第一个小于自身的柱体，那么就可以将栈顶柱体出栈来计算以其为高的矩形的面积了。

leetCode-198-打家劫舍

发表于 2020-05-23 分类于 数据结构与算法 ， leetcode
本文字数： 933 阅读时长 ≈ 1 分钟

题目描述

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，计算你 不触动警报装置的情况下 ，一夜之内能够偷窃到的最高金额。

示例 1:

输入: [1,2,3,1]

输出: 4

解释: 偷窃 1 号房屋 (金额 = 1) ，然后偷窃 3 号房屋 (金额 = >3)。
偷窃到的最高金额 = 1 + 3 = 4 。

示例 2:

输入: [2,7,9,3,1]

输出: 12

解释: 偷窃 1 号房屋 (金额 = 2), 偷窃 3 号房屋 (金额 = 9)，接着偷窃 5 号房屋 (金额 = 1)。
偷窃到的最高金额 = 2 + 9 + 1 = 12 。

解题思路

动态规划的经典问题，定义dp的状态为遍历到第i个元素时，前i个元素的最大不连续数字之和。

  • 如果偷第 i 个房屋的话，由于相邻的房屋不能偷，所以 dp[i] = dp[i - 2] + nums[i]
  • 如果不偷第 i 个房屋的话，那么 dp[i] = dp[i - 1]
  • 那么dp[i] = max(dp[i - 2] + nums[i],dp[i - 1])

leetCode-974-和可被K整除的子数组

发表于 2020-05-22 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

给定一个整数数组 A，返回其中元素之和可被 K 整除的（连续、非空）子数组的数目。

示例：

输入：A = [4,5,0,-2,-3,1], K = 5

输出：7

解释：

有 7 个子数组满足其元素之和可被 K = 5 整除：
[4, 5, 0, -2, -3, 1], [5], [5, 0], [5, 0, -2, -3], [0], [0, -2, -3], [-2, -3]

提示：

  • 1 <= A.length <= 30000
  • -10000 <= A[i] <= 10000
  • 2 <= K <= 10000

解题思路

最直接的思路就是穷举所有的情况，但是指数级别的时间复杂度，会超时。

第二种思路就是可以记录穷举时前面的元素里面的重复的和，但是也需要两层循环，在给定的数据范围也会超时。

还有一个思路是前缀和：

  • 假如数字a可被K整除，数字b对K取模的结果和a+b对K取模结果一定一样。
  • 假如给定数组[4,5,0,-2,-3,1]。
  • 数组中前n个元素和对5取模结果是[4,4,4,2,4,0]
  • 上面数组的特点是什么，如果两个前缀和mok K结果相等，就证明这两个差值为0，那么此时就有一个子数组符合条件。
  • 将取模的结果记录下来，下次和这个取模结果相同的中间部分的数组就是符合条件连续数组。

leetCode-287-寻找重复数

发表于 2020-05-21 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

题目描述

给定一个包含n + 1 个整数的数组nums，其数字都在 1 到 n之间（包括 1 和 n），可知至少存在一个重复的整数。假设只有一个重复的整数，找出这个重复的数。

示例 1:

输入: [1,3,4,2,2]

输出: 2

示例 2:

输入: [3,1,3,4,2]
输出: 3

说明：

  • 不能更改原数组（假设数组是只读的）。
  • 只能使用额外的 O(1) 的空间。
  • 时间复杂度小于 O(n2) 。
  • 数组中只有一个重复的数字，但它可能不止重复出现一次。

解题思路

题目给定大小为n + 1数组并且元素范围为在 1 到 n 之间，并且只有一个重复的数字。

  • 假如数组中有十个数字，并且重复的数字是3。那么重复数字和其他数字的关系是怎样的？
  • 因为除了3以外的数字都是不重复的，根据题目要求，数组中大于3的元素最多有m个(4~9之间，不可能重复最多为6)，而小于等于3的元素是(10 - m)个，m最多为6，所以小于等于3的元素最少为4个，数组中小于等于重复元素的个数严格大于这个重复元素。
  • 可以用二分法，取中间值，逐步缩小区间来找到要求的重复元素。
