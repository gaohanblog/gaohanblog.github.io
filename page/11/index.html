leetCode-690-员工的重要性

发表于 2020-01-24 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

题目描述

给定一个保存员工信息的数据结构，它包含了员工唯一的id，重要度 和 直系下属的id。

比如，员工1是员工2的领导，员工2是员工3的领导。他们相应的重要度为15, 10, 5。那么员工1的数据结构是[1, 15, [2]]，员工2的数据结构是[2, 10, [3]]，员工3的数据结构是[3, 5, []]。注意虽然员工3也是员工1的一个下属，但是由于并不是直系下属，因此没有体现在员工1的数据结构中。

现在输入一个公司的所有员工信息，以及单个员工id，返回这个员工和他所有下属的重要度之和。

示例 1:

输入: [[1, 5, [2, 3]], [2, 3, []], [3, 3, []]], 1

输出: 11

解释:
员工1自身的重要度是5，他有两个直系下属2和3，而且2和3的重要度均为3。因此员工1的总重要度是 5 + 3 + 3 = 11。

注意: 一个员工最多有一个直系领导，但是可以有多个直系下属
员工数量不超过2000。

解题思路

在给定的员工id中，这个员工有自己直属下属，但是这个员工的直属下属还有直属下属，所以是层层递减的关系，要计算出这个员工的重要度就需要加上所有下属的重要度。返回的重要度是要 和下属的下属的下属的….都要加起来,不单是直系。

美团校招笔试题-火星文字典

发表于 2020-01-23 分类于 数据结构与算法 ， 笔试题
本文字数： 3.7k 阅读时长 ≈ 3 分钟

题目描述

已知一种新的火星文的单词由英文字母（仅小写字母）组成，但是此火星文中的字母先后顺序未知。给出一组非空的火星文单词，且此组单词已经按火星文字典序进行好了排序（从小到大），请推断出此火星文中的字母先后顺序。

输入描述:

  • 一行文本，为一组按火星文字典序排序好的单词(单词两端无引号)，单词之间通过空格隔开

输出描述:

  • 按火星文字母顺序输出出现过的字母，字母之间无其他字符，如果无法确定顺序或者无合理的字母排序可能，请输出”invalid”(无需引号)

示例1:

输入:z x

输出:
zx

示例2:

输入:wrt wrf er ett rftt

输出:wertf

示例3

输入：z x z

输出:invalid

解题思路

题目中给两个重要条件：

  • 第一是给定的每个单词中，每个字符的顺序是没有经过字典排序的。
  • 第二是给定所有的单词是排序好了的。

所以可以根据前一个单词和当前单词来确定某些字符的字典序是多少，例如给定第一个单词wrt，第二个单词wrf,wrt优先级大于wrf。因为这两个单词的前两个字符相同，只有最后一个字符是不同的，所以字符t的优先级大于字符f。同样wrf和 er,可以确定字符中w优先级大于e。

这个题目和leetcode第269题是相同的，题目的思路就是遍历所有输入进去的单词，根据单词的序列，求两两单词中可以确定的字符的字典序，构造一个有向无环图。然后找到图中的拓扑序列就是所能确定的字典序。

美团校招笔试题-美团骑手包裹区间分组

发表于 2020-01-23 分类于 数据结构与算法 ， 笔试题
本文字数： 1.6k 阅读时长 ≈ 1 分钟

题目描述

题目描述

2110年美团外卖火星第3000号配送站点有26名骑手，分别以大写字母A-Z命名，因此可以称呼这些骑手为黄家骑士特工A，黄家骑士特工B…黄家骑士特工Z，某美团黑珍珠餐厅的外卖流水线上会顺序产出一组包裹，美团配送调度引擎已经将包裹分配到骑手，并在包裹上粘贴好骑手名称，如RETTEBTAE代表一组流水线包裹共9个，同时分配给了名字为A B E R T的5名骑手。请在不打乱流水线产出顺序的情况下，把这组包裹划分为尽可能多的片段，同一个骑手只会出现在其中的一个片段，返回一个表示每个包裹片段的长度的列表。

输入描述:

  • 输入数据只有一行，为一个字符串(不包含引号)，长度不超过1000，只包含大写字母’A’到’Z’，字符之间无空格。

输出描述:

  • 输出每个分割成片段的包裹组的长度，每个长度之间通过空格隔开

示例1

输入:

MPMPCPMCMDEFEGDEHINHKLIN

输出:
9 7 8

说明：

划分结果为MPMPCPMCM,DEFEGDE,HINHKLIN。

每个骑手最多出现在一个片段中。

像MPMPCPMCMDEFEGDE,HINHKLIN的划分是错误的，因为划分的片段数较少。

解题思路

题目意思就是求一个连续字符串中，可以分割为多少个不含有共同元素的子串。可以用滑动窗口的思想，定义一个左右指针，右指针从后向前遍历，找到和左指针所指定的相同元素，这样除了左右指定的区间的字串中含有当前指针指向的元素其他地方都没有。然后再去遍历这个区间里面第二个元素(做指针后挪一位)，右指针再从右向左遍历
。

一直到左指针走到字符串和确定的区间最后一个元素相等时，说明当前确定的区间就是最小的不和总串出现相同元素的子串。

输入描述:

输出描述:

  • 一共q行，每行一个整数代表喜好值为k的用户的个数

示例1

输入

输出:

样例解释:

  • 有5个用户，喜好值为分别为1、2、3、3、5，
  • 第一组询问对于标号[1,2]的用户喜好值为1的用户的个数是1
  • 第二组询问对于标号[2,4]的用户喜好值为5的用户的个数是0
  • 第三组询问对于标号[3,5]的用户喜好值为3的用户的个数是2

解题思路

第一开始思路就是直接暴力求解，用两层循环直接扫描原数组中的数据，但是给定的数据范围是n <= 300000,q<=300000，会超时。

也可以用dp的思想，定义一个二维dp[i][j]表示第i个用户之前(包括第i个用户)对于文章喜爱度为j的总人数。状态转移方程分两种情况，假如当前用户对于文章的喜爱值为j,就需要在前一个基础上加一即：dp[i][j] = dp[i - 1][j] + 1,根据dp状态定义需要设置i后面每个人的状态都需要先更新为dp[i][j]的值。但是题目给定的数据范围是n <= 300000,q<=300000，单单开辟一个二维的dp数组就是几十个G，肯定是通过不了的，只提供一种思路。

所以可以拿空间按换时间，把每个用户的喜好值作为键存入map中，对应的是用户的下标。这样，就可以根据用户的喜爱值找到对应所有的用户下标，再去排除不在范围内的用户的下标。这样做，基本上实现了O(n)的时间复杂度。

美团校招笔试题-字符串模式匹配

发表于 2020-01-22 分类于 数据结构与算法 ， 笔试题
本文字数： 2.2k 阅读时长 ≈ 2 分钟

题目描述

给出两个字符串，分别是模式串P和目标串T，判断模式串和目标串是否匹配，匹配输出 1，不匹配输出 0。模式串中‘？’可以匹配目标串中的任何字符，模式串中的 ’*’可以匹配目标串中的任何长度的串，模式串的其它字符必须和目标串的字符匹配。例如P=a?b，T=acb，则P 和 T 匹配。

输入描述:

  • 输入第一行包含一个字符串p， (1≤|p|≤20).
  • 输入第二行包含一个字符串t， (1≤|t|≤20).

输出描述:

  • 输出仅包含0和1的整数，0表示p和t不匹配，1表示p和t匹配。

示例1

输入:

s1 = a?b

s2 = ab

输出:0

示例2

输入:

s1 = a*b

s2 = ab

输出:1

示例3

输入:

s1 = a*b

s2 = a(cb

输出:
1

解题思路

这个题目和leetcode第10题有很多相似的地方，但也不完全行同。leetcode第十题对于*的描述是 匹配零个或多个前面的那一个元素，这个题目对于*的描述是可以匹配目标串中的任何长度的串。所以类似的题目也可以用类似的思路来所。

和第10题一样定义dp的状态为二维数组的dp[i][j] 表示目标串的前 i 个是否能被模式串的前 j个字符匹配。那么状态转移方程可以分两种情况讨论：

  • 第一种假如当前被遍历的s1的第i个字符和s2的第j个字符相等，或者s1的字符为?,说明这两个字符是可以匹配上的，所以他们的结果应该由前一个状态来决定，即dp[i][j] = dp[i - 1][j - 1]
  • 第二种就是假如被遍历的s1的字符为*,那么它可以匹配0个或无限个字符。
    • 当匹配0个的时候，例如s1 = [aa*] s2 = [aa],他的状态由i-1决定即dp[i][j] = dp[i - 1][j]
    • 当匹配多个的时候，例如s1 = [aa*] s2 = [aabcd]那么他应该由j - 1决定（无论s2后面有多少个字符都可以匹配上），即dp[i][j] = dp[i][j - 1]
    • 所以dp[i][j] = dp[i - 1][j] || dp[i][j - 1]
