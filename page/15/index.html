leetCode-51-八皇后问题

发表于 2019-12-07 分类于 数据结构与算法 ， leetcode
本文字数： 2.8k 阅读时长 ≈ 3 分钟

题目描述

n 皇后问题研究的是如何将 n 个皇后放置在 n×n 的棋盘上，并且使皇后彼此之间不能相互攻击。

给定一个整数 n，返回所有不同的 n 皇后问题的解决方案。

每一种解法包含一个明确的 n 皇后问题的棋子放置方案，该方案中 'Q' 和 '.' 分别代表了皇后和空位。

示例:

输入: 4

输出: [

[“.Q..”, // 解法 1

“…Q”,

“Q…”,

“..Q.”],

[“..Q.”, // 解法 2

“Q…”,

“…Q”,

“.Q..”]

]

解释: 4 皇后问题存在两个不同的解法。

解题思路

整理下回溯法整体的思路：

  • n*n的棋盘，当第一个棋子放在棋盘第一行第一列，则棋盘的第一列被攻击，当前棋子的格子到右下角被攻击
  • 第二个棋子，从左到右检查第二行不能被攻击的格子，放下棋子。
  • 第三个，和第二个棋子一样检查第三行不被上面两个棋子攻击的格子。
  • 假如第n个棋子他的第n行都被前面的攻击，说明前面棋子放的不可能有正确的结果集。那么回溯到上一层，继续向当前行后面遍历找到可以放的地方。
  • 假如我们遍历到最后一层找到可以放的格子，则保存到结果集。
  • 继续遍历，直到找不到不被攻击的格子，回溯到上一层。
  • 直到遍历玩每一行的最后一个元素，结束程序。

分析下时间的复杂度，放第一个皇后有N种放法，放第二个皇后不超过N(N-2),第三个皇后不超过N(N-2)(N-4)种放法，以此类推。总体上，时间复杂度为N的阶乘O(N!)。

题目描述。

题目描述。

编写一个函数，输入是一个无符号整数，返回其二进制表达式中数字位数为 ‘1’ 的个数（也被称为汉明重量）。

示例1:

输入：00000000000000000000000000001011

输出：3

解释：输入的二进制串

00000000000000000000000000001011 中，共有三位为 ‘1’。

示例 2：

输入：00000000000000000000000010000000

输出：1

解释：输入的二进制串

00000000000000000000000010000000 中，共有一位为 >’1’。

示例 3：

输入：11111111111111111111111111111101

输出：31

解释：输入的二进制串

11111111111111111111111111111101 中，共有 31 位为 ‘1’。

解题思路

第一种解法，因为1和数字n按位与操作，假如n二进制最后一位是1，则返回一，否则返回零。遍历每个二进制数字即可。

第二种解法，在二进制表示中，因为数字n和 n - 1相差1，所以数字 n 中最低位的 1 总是对应 n - 1 中的 0 。因此，将 n 和 n - 1 与运算总是能把 nn 中最低位的 1 变成 0 ，并保持其他位不变。

题目描述。

给定一个二维网格 board 和一个字典中的单词列表 words，找出所有同时在二维网格和字典中出现的单词。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母在一个单词中不允许被重复使用。

示例:

输入:

words = [“oath”,”pea”,”eat”,”rain”] and board =

[

[‘o’,’a’,’a’,’n’],

[‘e’,’t’,’a’,’e’],

[‘i’,’h’,’k’,’r’],

[‘i’,’f’,’l’,’v’]

输出: [“eat”,”oath”]

说明:

你可以假设所有输入都由小写字母 a-z 组成。

提示:

  • 你需要优化回溯算法以通过更大数据量的测试。你能否早点停止回溯？

解题思路

leetCode-208-实现Trie(字典树)

发表于 2019-12-04 分类于 数据结构与算法 ， leetcode
本文字数： 3k 阅读时长 ≈ 3 分钟

题目描述

实现一个 Trie (前缀树)，包含 insert, search, 和 startsWith 这三个操作。

示例:

Trie trie = new Trie();

trie.insert(“apple”);

trie.search(“apple”); // 返回 true

trie.search(“app”); // 返回 false

trie.startsWith(“app”); // 返回 true

trie.insert(“app”);

trie.search(“app”); // 返回 true

说明:

  • 你可以假设所有的输入都是由小写字母 a-z 构成的。
  • 保证所有输入均为非空字符串。

解题思路

字典树又称单词查找树，Trie树，是一种树形结构，是一种哈希树的变种。典型应用是用于统计，排序和保存大量的字符串（但不仅限于字符串），所以经常被搜索引擎系统用于文本词频统计。它的优点是：利用字符串的公共前缀来减少查询时间，最大限度地减少无谓的字符串比较，查询效率比哈希树高。

它有3个基本性质：

  • 根节点不包含字符，除根节点外每一个节点都只包含一个字符；
  • 从根节点到某一节点，路径上经过的字符连接起来，为该节点对应的字符串；
  • 每个节点的所有子节点包含的字符都不相同。

搜索字典项目的方法为：

  • (1) 从根结点开始一次搜索；
  • (2) 取得要查找关键词的第一个字母，并根据该字母选择对应的子树并转到该子树继续进行检索；
  • (3) 在相应的子树上，取得要查找关键词的第二个字母,并进一步选择对应的子树进行检索。
  • (4) 迭代过程……
  • (5) 在某个结点处，关键词的所有字母已被取出，则读取附在该结点上的信息，即完成查找。
    其他操作类似处理

上面是某百科给字典树的定义，了解了基本性质，该怎么定义这个树那？

  • 首先只含有一个根节点并且为空，所以在Tire定义一个成员变量root。
    • 除根节点外每一个节点都只包含一个字符，题目要求26个小写字母，所以需要根据根节点定义26个指针向下扩展树的结构，可以用一个数组来保存这26个指针。
    • 题目中要求实现search函数，意思就是在搜索到某个节点处假如和单词相同的话需要返回这个单词是存在的，即定义一个标志位，来返回当前节点和前面节点连起来是一个单词，下面是代码实现。

leetCode-36&37-解数独

发表于 2019-12-02 分类于 数据结构与算法 ， leetcode
本文字数： 2.5k 阅读时长 ≈ 2 分钟

题目描述

编写一个程序，通过已填充的空格来解决数独问题。

一个数独的解法需遵循如下规则：

数字 1-9 在每一行只能出现一次。

数字 1-9 在每一列只能出现一次。

数字 1-9 在每一个以粗实线分隔的 3x3 宫内只能出现一次。
空白格用 ‘.’ 表示。

Note:

给定的数独序列只包含数字 1-9 和字符 ‘.’ 。
你可以假设给定的数独只有唯一解。
给定数独永远是 9x9 形式的。

解题思路

解数独回溯法解法就是模拟人解数独时的想法，来分析下怎么思路

  • 当我们解数独时，按照题目的要求，每一行，每一列和每一九个单元格数字不能重复，那么当我们下第一步时，会观察这三个要求，假如都满足的话，在第一个格子上放上一个满足条件的数字。
  • 然后按照条件再放第二个数字、第三个数字……势必会出现一种情况，再放第n个数字的时候，发现一到九都不满足题目的要求条件，这时候可以更改前面的数字，看看会不会满足条件。
  • 在程序中，也是这个道理，从第一行第一列挨个放数字，当放到第n个时出现不满足条件的数字，即回溯到上一层，更改其他数字，一遍一遍尝试。
  • 如果全部的行都填充完，则成功解锁了数独。

leetCode-22-括号生成

发表于 2019-12-01 分类于 数据结构与算法 ， leetcode
本文字数： 2.6k 阅读时长 ≈ 2 分钟

题目描述

给出 n 代表生成括号的对数，请你写出一个函数，使其能够生成所有可能的并且有效的括号组合。

例如，给出 n = 3，生成结果为：

[

“((()))”,

“(()())”,

“(())()”,

“()(())”,

“()()()”

]。

解题思路

题意大概就是左括号和右括号在不同的数量下有多少种不同的排列组合的方式，但是要求是必须合法。

先去分析下，假如题意要求列举出3个括号的全部可能，第一个符号一定是括号的左半部分，第二个符号可以是左半部分，也可以是右半部分，假如是第二个符号是括号的右半部分，第三个符号一定是括号的左半部分。有了这个思路，我们用深度优先的方法去实现，就是找到出口条件就是当左右符号都用完，或者前面的左括号数量和右括号相同，这时候必须上右括号了有点回朔的思想。

这个题目还有一个经典解法，就是用到了动态规划思想。一般来说，只要问题可以划分为规模更小的子问题，并且原问题的最优解中包含了子问题的最优解，则可以考虑用动态规划解决。

先来考虑一个问题怎么得到这个最优解

  • i 对括号的一个组合，一定以左括号 “(“ 开始，不一定以 “)” 结尾。为此，我们可以枚举新的右括号 “)” 可能所处的位置，得到所有的组合；
  • 枚举的方式就是枚举左括号 “(“ 和右括号 “)” 中间可能的合法的括号对数，而剩下的合法的括号对数在与第一个左括号 “(“ 配对的右括号 “)” 的后面，这就用到了以前的状态。

所以我们得到一个前一个结果和后一个结果的状态转移方程：

dp[i] = "(" + dp[可能的括号对数] + ")" + dp[剩下的括号对数]

  • “可能的括号对数” 与 “剩下的括号对数” 之和得为 i - 1，故 “可能的括号对数” j 可以从 0 开始，最多不能超过 i， 即 i - 1；
  • “剩下的括号对数” + j = i - 1，故 “剩下的括号对数” = i - j - 1。

leetCode-152-乘积最大子序列

发表于 2019-11-28 分类于 数据结构与算法 ， leetcode
本文字数： 1.6k 阅读时长 ≈ 1 分钟

题目描述

给定一个整数数组 nums，找出一个序列中乘积最大的连续子序列（该序列至少包含一个数）。

示例 1:

输入: [2,3,-2,4]

输出: 6

解释:子数组 [2,3] 有最大乘积 6。

示例 2:

输入: [-2,0,-1]

输出: 0

解释:结果不能为 2, 因为 [-2,-1] 不是子数组。

解题思路

主要来找到先状态转移的方程，来理一下思路：

  • 假如全为正数，我们直接累乘得到结果即可。
  • 题目中有负数，假如碰到一个负数怎么办，先把前面的乘积保存起来。跳过负数再乘后面的数字。
  • 假如又碰到一个负数，负负得正，这时候最大的乘积肯定是全部累乘的结果。
  • 这题是求数组中子区间的最大乘积，只维护正数的乘积肯定是不行的，负数乘以负数，会变成正数，所以解这题的时候我们需要维护两个变量，当前的最大值，以及最小值。最小值可能为负数，但没准下一步乘以一个负数，当前的最大值就变成最小值，而最小值则变成最大值了。
  • 所以 假如第i个数字为正数dp_max[i] = dp_max[i - 1] * cur 但是假如第i-1状态是负数怎么办，需要更当前最大乘积为当前数字所以最终是dp_max[i] = Max(dp_max[i - 1] * cur,cur)
  • 考虑到负数情况需要一个最小值区间，dp_min[i] = Min(dp_min[i - 1] * cur,cur)
  • 所以 假如第i个数字为负数，负负得正，此时最小和最大交换dp_max[i] = Max(dp_min[i - 1] * cur,cur)，最小值同理和最大交换dp_min[i] = Min(dp_max[i - 1] * cur,cur)
