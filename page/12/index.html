本文字数： 2.9k 阅读时长 ≈ 3 分钟

题目描述

地铁迷在某个城市组织了地铁打卡活动。活动要求前往该城市中的所有地铁站进行打卡。打卡可以在站外或者站内进行。地铁的计价规则如下：只要不出站，就不计费；出站时，只计算进站和出站站点的距离。如在同一个站点进出站，按照最低票价 a 元计算。假设地铁票不会超时。大部分站点都是通过地铁线连通的，而且地铁站的连通是双向的（若 A，B 连通，则 B，A连通），且具有传递性的（若 A，B 连通，且 B，C 连通，则 A，C连通）。但并不是所有的地铁站都相互连通，在不能通过坐地铁达到的两个地点间，交通的花费则提高到 b 元。地铁迷从酒店起点出发，再回到酒店。假设从酒店到达任意地铁站的交通花费为 b 元。请计算地铁迷完成打卡最小交通花费。

输入描述:

  • 第2到m + 1行，每行2个整数Hi，Ti代表站点Hi和Ti站点是连通的（0 <= i< m）。

输出描述:

  • 输出只有一行，包含一个整数，表示打卡的最小交通花费。

示例1

输入

输出:24

一开始思路就是创建以一个二维连通图，去遍历这个图，找到连通区域有多少个，然后直接计算结果。

leetCode-17.16-按摩师

发表于 2020-01-22 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

一个有名的按摩师会收到源源不断的预约请求，每个预约都可以选择接或不接。在每次预约服务之间要有休息时间，因此她不能接受相邻的预约。给定一个预约请求序列，替按摩师找到最优的预约集合（总预约时间最长），返回总的分钟数。

示例 1：

输入： [1,2,3,1]

输出： 4

解释： 选择 1 号预约和 3 号预约，总时长 = 1 + 3 = 4。

示例 2：

输入： [2,7,9,3,1]

输出： 12

解释： 选择 1 号预约、 3 号预约和 5 号预约，总时长 = 2 + 9 + 1 = 12。

解题思路

典型的动态规划的问题,先找到dp的状态，dp的状态就是当一个预约过来的时候，计算出到当前预约的最大的收益：

  • 当第一个预约出现的时候，按摩师有两种选择接受和不接受，当按摩师接受，那么当前最大收益是dp[0] = 当前预约
  • 第二个预约，因为不能连着接受预约，如果按摩师结束当前预约就要放弃前一个预约，所以dp[1] = max(当前预约,前一个预约)
  • 第第三个预约，要么选择这个预约，放弃前一个预约，要么选择前一个预约状态下的最大收益,所以dp[2] = max(当前预约+不选择前一个,前一个预约)
  • 所以状态转移方程为dp[i] = max(dp[i-1], dp[i-2] + 当前预约收益)

leetCode-61-旋转链表

发表于 2020-01-22 分类于 数据结构与算法 ， leetcode
本文字数： 2.3k 阅读时长 ≈ 2 分钟

题目描述

给定一个链表，旋转链表，将链表每个节点向右移动 k 个位置，其中 k 是非负数。

示例 1:

输入: 1->2->3->4->5->NULL, k = 2

输出: 4->5->1->2->3->NULL

解释:

向右旋转 1 步: 5->1->2->3->4->NULL

向右旋转 2 步: 4->5->1->2->3->NULL

示例 2:

输入: 0->1->2->NULL, k = 4

输出: 2->0->1->NULL

解释:

向右旋转 1 步: 2->0->1->NULL

向右旋转 2 步: 1->2->0->NULL

向右旋转 3 步: 0->1->2->NULL

向右旋转 4 步: 2->0->1->NULL

解题思路

最开始思路很简单，因为链表每次移动都是将尾节指向头节点作为新的头节点，那么移动k次就是结果，但是结果超时了，因为假如移动10000次，就需要重新将链表调整10000遍，显然不行的，也是一种思路

顺着上面思路，既然向右移动k个位置，每次调整链表的结构很耗时，因为每次移动头节点都会向后移动一步，可以先求出来当移动k步以后，头节点所在的位置，找到头节点的位置。

  • 假如链表长度尾5，移动4步
  • 那么计算出移动后的头节点在最后一位
  • 那么新的头节点就是原来头节点的下一位。

这样做的好处是不用每次调整链表，只需要在最后一步找到头节点的位置，做一次链表的调整即可。

美团校招笔试题-表达式求值

发表于 2020-01-21 分类于 数据结构与算法 ， 笔试题
本文字数： 2k 阅读时长 ≈ 2 分钟

题目描述

给出一个布尔表达式的字符串，比如：true or false and false，表达式只包含true，false，and和or，现在要对这个表达式进行布尔求值，计算结果为真时输出true、为假时输出false，不合法的表达时输出error（比如：true true）。表达式求值是注意and 的优先级比 or 要高，比如：true or false and false，等价于 true or (false and false)，计算结果是 true。

输入描述:

  • 输入第一行包含布尔表达式字符串s，s只包含true、false、and、or几个单词（不会出现其它的任何单词），且单词之间用空格分隔。 (1≤|s|≤103).

输出描述:

  • 输出true、false或error，true表示布尔表达式计算为真，false表示布尔表达式计算为假，error表示一个不合法的表达式。

示例1

输入:and

输出:error

示例2

输入:true and false

输出:false

示例3

输入:true or false and false

输出
true

解题思路

先找到输入表达式的特点：

  • 第一个字符串和最后一个字符串必须为false或者true
  • 字符串中逻辑符(and和or)和布尔值(false和true)必须是间隔出现的。
  • 根据上面特点既然第一个输入的布尔值，如果字符串合法那么下一个一定逻辑符，所以可以说偶数位上的都是逻辑符，奇数位上的是布尔值
  • 所以可以通过依次遍历字符串查看是否合法

判断了字符串的合法性以后，就要返回结果了，根据and和or的不同特点，可以先把字符串中所有and的值计算出来，然后再计算or的值。

发表于 2020-01-21 分类于 数据结构与算法 ， leetcode

题目描述

给定一个带有头结点 head 的非空单链表，返回链表的中间结点。

如果有两个中间结点，则返回第二个中间结点。

示例 1：

输入：[1,2,3,4,5]

输出：此列表中的结点 3 (序列化形式：[3,4,5])

返回的结点值为 3 。(测评系统对该结点序列化表述是[3,4,5])。

注意，我们返回了一个 ListNode 类型的对象 ans，这样：
ans.val = 3, ans.next.val = 4, ans.next.next.val = 5, 以及 ans.next.next.next = NULL.

示例 2：

输入：[1,2,3,4,5,6]

输出：此列表中的结点 4 (序列化形式：[4,5,6])
由于该列表有两个中间结点，值分别为 3 和 4，我们返回第二个结点。

解题思路

第一种思路就是直接遍历链表计算链表节点总共的数量，然后在从头遍历节点，然后找到中间节点。

还有一种比较好的思路就是快慢指针，快指针一次走两步，慢指针一次走一步，当快指针走到尾巴时，此时慢指针正好在中间节点。

题目描述

给定整数数组 A，每次 move 操作将会选择任意 A[i]，并将其递增 1。

返回使 A 中的每个值都是唯一的最少操作次数。

示例 1:

输入：[1,2,2]

输出：1

解释：经过一次 move 操作，数组将变为 [1, 2, 3]。

示例 2:

输入：[3,2,1,2,1,7]

输出：6

解释：经过 6 次 move 操作，数组将变为 [3, 4, 1, 2, 5, 7]。

可以看出 5 次或 5 次以下的 move

操作是不能让数组的每个值唯一的。

解题思路

第一种思路就是直接排序，再依次遍历数组元素，若当前元素小于等于它前一个元素，则将其变为前一个数+1。

第二种思路是,题目中规定了A[i]的长度< 40000，可以定义一个40000长度的数组，把原数组中的所有数字作为下标，把每个数作为值，然后把数组中值不为1的元素执行+1操作。

leetCode-409-最长回文串

发表于 2020-01-18 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

给定一个包含大写字母和小写字母的字符串，找到通过这些字母构造成的最长的回文串。

在构造过程中，请注意区分大小写。比如 “Aa” 不能当做一个回文字符串。

示例 1:

输入:
“abccccdd”

输出:
7

解释:
我们可以构造的最长的回文串是”dccaccd”, 它的长度是 7。

解题思路

回文字符串的特点是正着读和反着读都一样的字符串，那么回文字符串中的不同字符的个数必然是偶数个，题目要求找到最大回文字符串，只要找到所有偶数位的字符就可以。例如字符串 “abba” 中，回文中心是 “ab|ba” 中竖线的位置，而在字符串 “abcba” 中，回文中心是 “ab(c)ba” 中的字符 “c” 本身。我们可以发现，在一个回文串中，只有最多一个字符出现了奇数次，其余的字符都出现偶数次。
