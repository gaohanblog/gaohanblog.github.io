leetCode-958-二叉树的完全性检验

发表于 2020-03-20 更新于 2019-03-20 分类于 数据结构与算法 ， leetcode
本文字数： 1.6k 阅读时长 ≈ 1 分钟

题目描述

给定一个二叉树，确定它是否是一个完全二叉树。

百度百科中对完全二叉树的定义如下：

若设二叉树的深度为 h，除第 h 层外，其它各层 (1～h-1) 的结点数都达到最大个数，第 h 层所有的结点都连续集中在最左边，这就是完全二叉树。（注：第 h 层可能包含 1~ 2h 个节点。）

示例 1:

输入：[1,2,3,4,5,null,7]

输出：false

解释：值为 7 的结点没有尽可能靠向左侧。

示例 2:

输入：[1,2,3,4,5,null,7]

输出：false

解释：值为 7 的结点没有尽可能靠向左侧。

提示：

树中将会有 1 到 100 个结点。

解题思路

根据完全二叉树的特点，用层次遍历是去遍历树，然后根据讨论在遍历过程中会出现不符合完全二叉树的情况：

  • 第一种情况，当在遍历过程中，一个节点的左节点为空，右节点不为空，那么一定不是完全二叉树。
  • 如果在遍历过程中出现了节点的左节点或者右节点中有空节点，假如是完全二叉树说明下一层深度的树就是树的最后一层。
  • 当找到了节点的左节点或者右节点中有空节点，那么后面再进入队列的节点的左节点和右节点必须为叶子节点，否则不满足完全二叉树的特点。

leetCode-649-Dota2参议院

发表于 2020-03-19 更新于 2019-03-19 分类于 数据结构与算法 ， leetcode
本文字数： 2.1k 阅读时长 ≈ 2 分钟

题目描述

Dota2 的世界里有两个阵营：Radiant(天辉)和 Dire(夜魇)

Dota2 参议院由来自两派的参议员组成。现在参议院希望对一个 Dota2 游戏里的改变作出决定。他们以一个基于轮为过程的投票进行。在每一轮中，每一位参议员都可以行使两项权利中的一项：

禁止一名参议员的权利：

参议员可以让另一位参议员在这一轮和随后的几轮中丧失所有的权利。

宣布胜利：

如果参议员发现有权利投票的参议员都是同一个阵营的，他可以宣布胜利并决定在游戏中的有关变化。

给定一个字符串代表每个参议员的阵营。字母 “R” 和 “D” 分别代表了Radiant（天辉）和Dire（夜魇）。然后，如果有 n 个参议员，给定字符串的大小将是 n。

以轮为基础的过程从给定顺序的第一个参议员开始到最后一个参议员结束。这一过程将持续到投票结束。所有失去权利的参议员将在过程中被跳过。

假设每一位参议员都足够聪明，会为自己的政党做出最好的策略，你需要预测哪一方最终会宣布胜利并在 Dota2 游戏中决定改变。输出应该是Radiant或Dire。

示例 1:

输入: “RD”

输出: “Radiant”

解释: 第一个参议员来自 Radiant >阵营并且他可以使用第一项权利让第二个参议员失去权力，因此第>二个参议员将被跳过因为他没有任何权利。然后在第二轮的时候，>第一个参议员可以宣布胜利，因为他是唯一一个有投票权的人

示例 2:

输入: “RDD”

输出: “Dire”

解释:
第一轮中,第一个来自 Radiant >阵营的参议员可以使用第一项权利禁止第二个参议员的权利
第二个来自 Dire 阵营的参议员会被跳过因为他的权利被禁止
第三个来自 Dire >阵营的参议员可以使用他的第一项权利禁止第一个参议员的权利
因此在第二轮只剩下第三个参议员拥有投票的权利,于是他可以宣布胜利

注意:

给定字符串的长度在 [1, 10,000] 之间.

解题思路

这个题目可以用贪心的思想来解决，下面分析一下：

  • 假如遍历到第n个议员属于Radiant阵营并且没有被禁止的话，那么他要做的就是禁止下一位发言的Dire阵营的议员，因为执行权力的先后顺序决定着输赢。所以应该先把首先使用权力的最近的一个不同阵营的人禁言掉。
  • 在代码中，不用在遍历到D或者R的时候,直接去后面找到不同阵营的人，可以先作为一个变量记下来，继续遍历，当出现了不同阵营的人要发言的时候在禁止掉。

leetCode-542-01矩阵

发表于 2020-03-18 更新于 2019-03-18 分类于 数据结构与算法 ， leetcode
本文字数： 2k 阅读时长 ≈ 2 分钟

题目描述

给定一个由 0 和 1 组成的矩阵，找出每个元素到最近的 0 的距离。

两个相邻元素间的距离为 1 。

示例 1:

输入:

0 0 0
0 1 0
0 0 0

输出:

0 0 0
0 1 0
0 0 0

示例 2:

输入：

0 0 0
0 1 0
1 1 1

输出：

0 0 0
0 1 0
1 2 1

解题思路

矩阵类型的题目，一般都是DFS或者BFS，一开始的想的用DFS遍历图中的1，用1向四周扩散，找到四周的0和1，如果有0则返回1，没有则继续对这个点做DFS，因为图中至少有一个0，所以一定存在终点。可是这样做似乎是错的，在做深度遍历的时候，假如有两个点连接的点1。

  • 先看下深度遍历目的是返回周围点的最小到达0的距离。
  • 当遍历到第一个点1时，需要向上下左右四周做DFS，可是假如上下左右还有一个点是1，再对这个点做深度遍历的时候，还需要用到原来那个点1，但是原来点1的距离根本没有求出来，所以形成相互依赖的死循环。

所以无论BFS还是DFS都不可以先去遍历点1，需要逆推，就是从点0出发，先把所有点0周围为1的距离定为1，这样剩下的点可以求距离的时候就可以直接从原来求出距离的点加上1就可以了。

leetCode-445-两数相加II

题目描述

给你两个 非空 链表来代表两个非负整数。数字最高位位于链表开始位置。它们的每个节点只存储一位数字。将这两数相加会返回一个新的链表。

你可以假设除了数字 0 之外，这两个数字都不会以零开头。

进阶：

如果输入链表不能修改该如何处理？换句话说，你不能对列表中的节点进行翻转。

示例 1：

输入：(7 -> 2 -> 4 -> 3) + (5 -> 6 -> 4)

输出：7 -> 8 -> 0 -> 7

解题思路

最直接的思路就是直接反转链表，然后挨个相加，但是题目说尽量不要用链表的反转，所以还是想想其他办法吧。

还有一种思路就是栈，不让反转，可以把链表元素都装入栈中，同样也实现了链表的倒序相加，遍历元素时使用头插法添加节点。

leetCode-23-合并K个排序链表

发表于 2020-03-16 更新于 2019-03-16 分类于 数据结构与算法 ， leetcode
本文字数： 3.4k 阅读时长 ≈ 3 分钟

题目描述

合并 k 个排序链表，返回合并后的排序链表。请分析和描述算法的复杂度。

示例 1：

输入:
[
1->4->5,
1->3->4,
2->6
]

输出: 1->1->2->3->4->4->5->6

解题思路

合并n个有序链表，第一种思路就是直接暴力遍历数组中所有的节点，将第一个和第二个合并，再将第三个与前面的结果合并，然后第四个…..

假如每个链表的长度为k，数组长度为n

  • 遍历数组的时间是O(n)的。
  • 当链表1和链表2合并为新链表时，都需要至少比较k次才可以(k为两个链表的长度)所以为O(k)。
  • 所以总体时间复杂度是O(k*n)的。
  • 空间复杂度上面，再计算中没有重新开辟新的空间，只是改变了原来链表节点的指向，所以是O(1)的。

有了上面遍历数组每次都一次只合并一个链表，可不可以用快排的思想，先将数组分开然后一次合并两个链表。时间复杂度上面类似于快排的时间。

  • 和上面一样，两个链表合并时比较节点个数是O(k)次的。
  • 当采用二分时将数组中n个链表链表的进程加起来是O(logn)次的
  • 所以时间复杂度是O(k*logn).
  • 空间复杂度上面，和上面一样，也是O(1)的。

还有一种思路就是用小顶堆，先把所有的节点都添加到小顶堆中，那么小顶堆的堆顶元素就是目前的最小节点了，找到这个最小点以后，更新堆里面的元素，将刚刚那么最小点的下一个节点在加入到堆中，和其他元素比较。一直到数组中所有的链表都遍历完返回即可。

  • 小顶堆里面有n个元素，小顶堆每次找到最小点是O(1)的时间，堆顶元素出堆和下一个元素进入小顶堆都需要log(n)的时间。
  • 所有链表都有k个节点。
  • 所以总的时间复杂度是O(k*logn)的
  • 空间复杂度上面，和上面一样链表结构的调整，只是改变了链表的指向，所以为O(1)的，但是需要创建一个大小和数组长度相等的优先队列，所以O(n)的开销。

leetCode-540-有序数组中的单一元素

发表于 2020-03-15 更新于 2019-03-15 分类于 数据结构与算法 ， leetcode
本文字数： 1.9k 阅读时长 ≈ 2 分钟

题目描述

给定一个只包含整数的有序数组，每个元素都会出现两次，唯有一个数只会出现一次，找出这个数。

示例 1：

输入: [1,1,2,3,3,4,4,8,8]

输出: 2

示例 2:

输入: [3,3,7,7,10,11,11]

输出: 10

注意: 您的方案应该在 O(log n)时间复杂度和 O(1)空间复杂度中运行。

解题思路

假如没有题目要求必须用二分法，可以直接遍历数组，只存在一个数出现一次的情况，遍历过程中数组下标为奇数和为偶数应该都相同，不同则证明这个奇数位就是要求的数字。

下面看一下二分的主要思路：

  • 数组中每个元素都会出现两次，唯有一个数只会出现一次，所以数组的元素个数是奇数个的。所以二分的时候，mid左右两部分元素的个数是相等的。
  • 那么就以找到的中点下标记为mid，左右两部分的个数可能为偶数，也可能为奇数。
    • 当为奇数个时，如果左半部分最后一个值和中间值相等时，那么左半部分加上中间值正好构成了偶数对数字，所以这时左半部分不可能出现单个数字的情况。
    • 当为偶数个时，可能出现两种情况：
      • 第一种时，这个中间值就是单独的数字，即左右两部分都构成了成对出现的数字。这种情况下中间值和左右相邻的数字都不构成对子。
      • 当中间值和左右相邻的元素有一个构成了对子，比如和左半部分最后一个数字构成了对子，说明左半部分中有一个数字是不能和前后相邻数字构成对子的，这时候再去二分查找左半部分即可。

leetCode-176-第二高的薪水

发表于 2020-03-14 更新于 2019-03-14 分类于 数据结构与算法 ， leetcode
本文字数： 541 阅读时长 ≈ 1 分钟

题目描述

编写一个 SQL 查询，获取 Employee 表中第二高的薪水（Salary） 。

示例 1：

Id Salary
1 100
2 200
3 300
例如上述 Employee 表，SQL查询应该返回 200 作为第二高的薪水。如果不存在第二高的薪水，那么查询应返回 null。

返回 SecondHighestSalary 为200

解题思路

这个问题主要考察了sql语句的ifnull函数，order by排序，distinct关键字,和limit分页使用。

  • ifnull用于判断第一个表达式是否为 NULL，如果为 NULL 则返回第二个参数的值，如果不为 NULL 则返回第一个参数的值，有点类似于三元表达式。
  • distinct 去掉重复的薪资。
  • order by salary desc 是根据薪资倒序倒序。
  • limit 1,1 是分页去除第二条。

leetCode-1318-或运算的最小翻转次数

发表于 2020-03-12 更新于 2019-03-12 分类于 数据结构与算法 ， leetcode
本文字数： 1k 阅读时长 ≈ 1 分钟

题目描述

给你三个正整数 a、b 和 c。

你可以对 a 和 b 的二进制表示进行位翻转操作，返回能够使按位或运算 a OR b == c 成立的最小翻转次数。

「位翻转操作」是指将一个数的二进制表示任何单个位上的 1 变成 0 或者 0 变成 1 。

示例 1：

输入：a = 2, b = 6, c = 5

输出：3

解释：翻转后 a = 1 , b = 4 , c = 5 使得 a OR b == c

解题思路

a,b,c都是int类型的数字，都有32位，三个数字逐位比较。

  • 先取到每个数字的最后一位，a & 1 = bitA 表示最后一位是0还是1
  • 得到每一数字最后一位，需要满足题目要求，bitA | bitB = bitC
    • bitC为1时,bitA 和bitB至少一位为1
    • bitC位0时，bitA和bitB 都得为零

leetCode-944-删列造序

发表于 2020-03-11 更新于 2019-03-11 分类于 数据结构与算法 ， leetcode
本文字数： 1.2k 阅读时长 ≈ 1 分钟

题目描述

给定由 N 个小写字母字符串组成的数组 A，其中每个字符串长度相等。

删除 操作的定义是：选出一组要删掉的列，删去 A 中对应列中的所有字符，形式上，第 n 列为 [A[0][n], A[1][n], …, A[A.length-1][n]]）。

比如，有A = [“abcdef”, “uvwxyz”]，

a b c d e f
u v w x y z

要删掉的列为 {0, 2, 3}，删除后 A 为[“bef”, “vyz”]， A 的列分别为[“b”,”v”], [“e”,”y”], [“f”,”z”]。

==a== b ==c== ==d== e f
==u== v ==w== ==x== y z

你需要选出一组要删掉的列 D，对 A 执行删除操作，使 A 中剩余的每一列都是 非降序 排列的，然后请你返回 D.length 的最小可能值。

示例 1：

输入：[“cba”, “daf”, “ghi”]

输出：1

解释：

当选择 D = {1}，删除后 A 的列为：[“c”,”d”,”g”] 和 [“a”,”f”,”i”]，均为非降序排列。

若选择 D = {}，那么 A 的列 [“b”,”a”,”h”] 就不是非降序排列了。

示例 3：

输入：[“zyx”, “wvu”, “tsr”]

输出：3

解释：D = {0, 1, 2}

解题思路

题目说的很长，看着很吓人，实际上对于数组每一列，我们检查它是否是有序的。如果它有序，则将答案增加 1，否则它必须被删除。
