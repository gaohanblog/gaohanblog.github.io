leetCode-287-寻找重复数

发表于 2020-05-21 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

题目描述

给定一个包含n + 1 个整数的数组nums，其数字都在 1 到 n之间（包括 1 和 n），可知至少存在一个重复的整数。假设只有一个重复的整数，找出这个重复的数。

示例 1:

输入: [1,3,4,2,2]

输出: 2

示例 2:

输入: [3,1,3,4,2]
输出: 3

说明：

  • 不能更改原数组（假设数组是只读的）。
  • 只能使用额外的 O(1) 的空间。
  • 时间复杂度小于 O(n2) 。
  • 数组中只有一个重复的数字，但它可能不止重复出现一次。

解题思路

题目给定大小为n + 1数组并且元素范围为在 1 到 n 之间，并且只有一个重复的数字。

  • 假如数组中有十个数字，并且重复的数字是3。那么重复数字和其他数字的关系是怎样的？
  • 因为除了3以外的数字都是不重复的，根据题目要求，数组中大于3的元素最多有m个(4~9之间，不可能重复最多为6)，而小于等于3的元素是(10 - m)个，m最多为6，所以小于等于3的元素最少为4个，数组中小于等于重复元素的个数严格大于这个重复元素。
  • 可以用二分法，取中间值，逐步缩小区间来找到要求的重复元素。

leetCode-146-LRU缓存机制

发表于 2020-05-20 分类于 数据结构与算法 ， leetcode
本文字数： 3.6k 阅读时长 ≈ 3 分钟

题目描述

运用你所掌握的数据结构，设计和实现一个 LRU (最近最少使用) 缓存机制。它应该支持以下操作： 获取数据 get 和 写入数据 put 。

获取数据 get(key) - 如果密钥 (key) 存在于缓存中，则获取密钥的值（总是正数），否则返回 -1。
写入数据 put(key, value) - 如果密钥已经存在，则变更其数据值；如果密钥不存在，则插入该组「密钥/数据值」。当缓存容量达到上限时，它应该在写入新数据之前删除最久未使用的数据值，从而为新的数据值留出空间。

进阶:

你是否可以在 O(1) 时间复杂度内完成这两种操作？

解题思路

主要思路就是用一个哈希表和一个双向链表维护所有在缓存中的键值对。

  • 双向链表的尾节点是保存缓存中最近使用的节点，头节点是最不常用的节点。
  • 哈希表键存储缓存的键，值为双向链表的一个节点。

一开始的时候看官方题解里面用到上面的思路，但是官方题解中使用的头插法，头节点保存最常用的一个节点，尾部节点是最不常用的节点，下面题解是用的尾插法，就是新访问和创建的节点移动到链表的尾部，头节点是最不常用的一个节点。

leetCode-4-寻找两个正序数组的中位数

发表于 2020-05-19 分类于 数据结构与算法 ， leetcode
本文字数： 1.6k 阅读时长 ≈ 1 分钟

题目描述

给定两个大小为 m 和 n 的正序（从小到大）数组nums1 和nums2。

请你找出这两个正序数组的中位数，并且要求算法的时间复杂度为O(log(m + n))。

你可以假设nums1和nums2不会同时为空。

示例 1:

nums1 = [1, 3]

nums2 = [2]

则中位数是 2.0

示例 2:

nums1 = [1, 2]

nums2 = [3, 4]

则中位数是 (2 + 3)/2 = 2.5

解题思路

题目求两个升序数组的中位数，最简单的思路就是将这两个数组合并，找到合并后数组中间的下标就是题目要求的中位数。但是当合并后数组的长度是偶数时，中位数是中间两个下标数字的平均数。数组长度是奇数，则中位数是中间的下标的数字。

上面那样做需要数组的合并，必然会带来额外空间的开销。题目中给定的条件是两个升序的数组，所以可以分别遍历两个数组(分别比两个数组每个元素的大小，相当于按照升序来遍历两个数组)，这样做只需要遍历到两个数组长度和的一半时就是要求的中位数。

leetCode-1371-每个元音包含偶数次的最长子字符串

发表于 2020-05-15 分类于 数据结构与算法 ， leetcode
本文字数： 1.9k 阅读时长 ≈ 2 分钟

题目描述

给你一个字符串 s ，请你返回满足以下条件的最长子字符串的长度：每个元音字母，即 ‘a’，’e’，’i’，’o’，’u’ ，在子字符串中都恰好出现了偶数次。

示例 1:

输入：s = “eleetminicoworoep”

输出：13

解释：最长子字符串是 “leetminicowor” ，它包含 e，i，o 各 2 >个，以及 0 个 a，u 。

示例 2:

输入：s = “leetcodeisgreat”

输出：5

解释：最长子字符串是 “leetc” ，其中包含 2 个 e 。

示例 3:

输入：s = “bcbcbc”

输出：6

解释：这个示例中，字符串 “bcbcbc” >
本身就是最长的，因为所有的元音 a，e，i，o，u 都出现了 0 次。

提示：

  • 1 <= s.length <= 5 x 10^5
  • s 只包含小写英文字母。

解题思路

最一开始思路是用滑动窗口，但是元音字母出现可能和前面的元音字母配对，说明不把右指针走完的情况下是不能确定前面的元音字母是否可以配对，所以不能解决问题。

还有一种思路在给定的字符串中所有的字串里面的元音字母最多有32中情况,用数字二进制’00000’每一位来表示字串里面五个元音字母(a,e,i,o,u)出现的是否为偶数或者奇数的情况。例如：

  • 给定的字符串是”eoleo”。
  • 当遍历第一个字符时候是e，是元音字母，对应00000中的第4位，这时候e是第一次出现，所以记录为00010。
  • 当遍历第二个字符时候是0,是元音字母，记录为01010。
  • 当遍历第三个字符时候是l,不是元音字母,不用记录。
  • 当遍历第四个字符时候是e,是元音字母,这时候e出现的次数为偶数，所以记录为01000。
  • 当遍历第五个字符时候是o,是元音字母,这时候0出现的次数为偶数，所以记录为00000。

根据上面的情况，在遍历字符串时，可以确定遍历到的字串是否为包含偶数次元音的字符串。怎么求出最优解，最优解可能存在的情况：
假如当遍历到第i个字符时记录偶数位为01000说明前面字符串只出现字符o没有匹配到偶数位，当遍历到第j个字符时还是01000说明j - i这个区间是满足条件的。

记录位公有有32种情况。定义一个数组存放遍历到当前字符时前面各个元音出现的奇偶状态，如果当前状态和前面某一个状态相等，说明区间中满足题目要求，更新最大值。

leetCode-20-有效的括号

发表于 2020-05-14 分类于 数据结构与算法 ， leetcode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目描述

给定一个只包括 ‘(‘，’)’，’{‘，’}’，’[‘，’]’ 的字符串，判断字符串是否有效。

有效字符串需满足：

左括号必须用相同类型的右括号闭合。
左括号必须以正确的顺序闭合。
注意空字符串可被认为是有效字符串。

示例 1:

输入: “()”
输出: true

示例 2:

输入: “()[]{}”
输出: true

示例 3:

输入: “{[]}”
输出: true

解题思路

典型用堆栈来解决的问题，题目要求主要是左右对称，主要考虑边界情况，如第一个入栈的必须为符号的左半部分，空字符串，奇数个符号的情况，不用进入循环，直接返回
入栈时，先判断，碰到一对满足的括号，例如（）{} ，就弹出；不满足就入栈，空栈也入栈
若最后栈为空，代表括号是有效的。

leetCode-220-课程表II

发表于 2020-05-11 分类于 数据结构与算法 ， leetcode
本文字数： 2k 阅读时长 ≈ 2 分钟

题目描述

现在你总共有 n 门课需要选，记为0到n-1。

在选修某些课程之前需要一些先修课程。例如，想要学习课程 0 ，你需要先完成课程1 ，我们用一个匹配来表示他们: [0,1]

给定课程总量以及它们的先决条件，返回你为了学完所有课程所安排的学习顺序。

可能会有多个正确的顺序，你只要返回一种就可以了。如果不可能完成所有课程，返回一个空数组。

说明：

  • 输入的先决条件是由边缘列表表示的图形，而不是邻接矩阵。详情请参见图的表示法。
  • 你可以假定输入的先决条件中没有重复的边。 [-1e7, 1e7]。

    示例1:

    输入: 2, [[1,0]]

    输出: [0,1]

    解释: 总共有 2 门课程。要学习课程 1，你需要先完成课程 >0。因此，正确的课程顺序为 [0,1] 。

    输出:

    输入: 4, [[1,0],[2,0],[3,1],[3,2]]

    输出: [0,1,2,3] or [0,2,1,3]

    解释:总共有 4 门课程。要学习课程 3，你应该先完成课程 1 和课程 2。并且课程 1 和课程 2 都应该排在课程 0 之后。因此，一个正确的课程顺序是[0,1,2,3] 。另一个正确的排序是[0,2,1,3] 。

提示:

这个问题相当于查找一个循环是否存在于有向图中。如果存在循环，则不存在拓扑排序，因此不可能选取所有课程进行学习。

通过 DFS 进行拓扑排序 - 一个关于Coursera的精彩视频教程（21分钟），介绍拓扑排序的基本概念。

拓扑排序也可以通过 BFS 完成。

解题思路

经典的有向无环图的拓扑排序，总体思路就是根据给定的课程依赖关系，可以构建一个邻接矩阵做拓扑排序。也可以直接根据题目中给定的课程关系，直接确定每个节点的入度，这样做好处是不再需要创建二维的邻接矩阵，省去空间开销。

leetCode-560-和为K的子数组

发表于 2020-05-10 分类于 数据结构与算法 ， leetcode
本文字数： 1.4k 阅读时长 ≈ 1 分钟

题目描述

给定一个整数数组和一个整数 k，你需要找到该数组中和为 k 的连续的子数组的个数。

说明：

  • 数组的长度为 [1, 20,000]。
  • 数组中元素的范围是 [-1000, 1000] ，且整数 k 的范围是 [-1e7, 1e7]。

    示例1:
    输入:nums = [1,1,1], k = 2

    输出: 2 , [1,1] 与 [1,1] 为两种不同的情况。

解题思路

一开始思路是滑动窗口，但是滑动窗口的前提是所有数字都是正数，题目是有负数的，所以滑动窗口是不可以的。

如果题目中出现负数，就说明每个子数组的和都可能是k，所以最暴力的解法就是以直接查找以i开头j结尾的数组中有多少个符合条件的解。

遍历数组nums，计算从第0个元素到当前元素的和，用哈希表保存出现过的累积和sum的次数。如果sum - k在哈希表中出现过，则代表从当前下标i往前有连续的子数组的和为sum，和上面相比降低了时间复杂度。
