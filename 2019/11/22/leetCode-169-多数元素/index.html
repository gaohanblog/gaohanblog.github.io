leetCode-169-多数元素

发表于 2019-11-22 分类于 数据结构与算法 ， leetcode
本文字数： 2.1k 阅读时长 ≈ 2 分钟

题目描述

给定一个大小为 n 的数组，找到其中的多数元素。多数元素是指在数组中出现次数大于 ⌊ n/2 ⌋ 的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

示例1 ：

输入: [3,2,3]

输出: 3

示例 2:

输入: [2,2,1,1,1,2,2]

输出: 2

解题思路

最暴力的解法就是排序，然后遍历数组找到出现次数最多的数字。需要排序，所以时间复杂度是O(NlogN)的。

第二种解法是利用hashmap，遍历一遍数组，把所有的元素都添加到map里，然后找到map里面value最大的数字返回即可。map的查询是常数级别的，所以这种解法时间复杂度是O(N)的，但是需要额外的空间。

还可以利用分治的思想，把数组从中间分开二分查找，逐渐返回出左右两边数字出现最多的一个，题目中说明了数字出现的次数一定大于 ⌊ n/2 ⌋ 的元素，所以必定返回这个数字。这种算法的时间复杂度，因为用到二分，并且数组中所有的数字都要遍历一遍，所以时间复杂度是(NlogN)

还有一种特别巧妙的解法，被称为摩尔投票法先假设第一个数过半数并设count=1；遍历后面的数如果相同则count+1，不同则减一，当count为0时则更换新的数字为候选数（成立前提：有出现次数大于n/2的数存在）,这种巧妙的解法，不需要额外的空间，而且只需要O(N)的时间,所以目前来讲是最好的解法了。
