Lambda表达式

Lambda 表达式，也可称为闭包，它是推动 Java 8 发布的最重要新特性。Lambda 允许把函数作为一个方法的参数（函数作为参数传递进方法中）先从一个小例子看一下来看一下Lambda的语法。

#Lambda表达式的语法解析

Lambda表达式在Java语言中引入了 “->” 操作符， “->” 操作符被称为Lambda表达式的操作符或者箭头操作符，它将Lambda表达式分为两部分：

  • 左侧部分指定了Lambda表达式需要的所有参数。

Lambda表达式本质上是对接口的实现，Lambda表达式的参数列表本质上对应着接口中方法的参数列表。

  • 右侧部分指定了Lambda体，即Lambda表达式要执行的功能。

下面看一下JDK8中提供的函数式接口，如何对于Lambda表达式的支持的。

#函数式接口

  • JDK8新增的接口注解@FunctionalInterface

    • 在接口中可以添加使用 default 关键字修饰的非抽象方法。即：默认方法（或扩展方法）

    • 接口里可以声明静态方法，并且可以实现。

    • 接口中只能有一个抽象方法

为什么@FunctionalInterface会这样要求那和Lambda表达式能产生怎样的化学反应。

JDK8在java.util.function包下内置了多数常用的函数式接口，其中根据参数类型和返回类型主要有以下常用的几个内置函数：

函数式接口 参数类型 返回类型 实现方法 用途
Consumer<T> 消费型接口 T 无返回值 void accept(T t) 主要对传入类型T进行操作
Supplier<T>供给型接口 无参数 T T get() 返回T类型的对象
Function<T,R>函数型接口 T R R apply(T t) 传入T为对象，返回R类型的对象
Predicate<T>断定型接口 T boolean boolean test(T t) 传入T为对象确保方法条件是否满足
UnaryOperator<T>一元函数 T T T apply(T t)继承自Function 输入和输出类型相同
BiFunction<T, U, R>两个输入函数 T U R R apply(T t, U u) 传入两个类型，返回一个类型
BinaryOperator<T>二元函数 T T T T apply(T t, T u)继承自BiFunction 输入和输出类型相同

对于上面给定的接口，配合Lambda的语法，可以使代码特别清爽简单。

#什么是Stream流

关于流再知乎上找到某大佬的描述，很是生动,如下:

Stream 不是集合元素，它不是数据结构并不保存数据，它是有关算法和计算的，如果把流比作一个工厂的生产线，生产线上有工人（或者说机器人）。每个工人一般都是在一个固定的位置，对处于某个状态的产品按指定的工序进行加工。那么，把生产线上的一个接一个的产品当成流（Stream），工人就是流处理器（Stream Wrapper/Filter，即加工或筛选），然后生产线上运输产品的装置（比如传送带）就是管道（Pipeline）。

流处理就是将数据处理模拟成工厂的生产线，每一道工序（wrapper）上都有特定的处理程序，处理好了就pipe到下一道工序。用伪代码表示如下：

a.pipe(b).pipe(c).pipe(d); // ....

所以说流（Stream）是指数据（Data），而不是容器。流的容器是管道（Pipeline），即运输装置。流的本意是模拟水流的特性，水流从上游流动到下游，编程里的流也是一样的。对于单个流处理器（工序）而言，上一个流处理器（工序）就是上游，下一个就是下游。
