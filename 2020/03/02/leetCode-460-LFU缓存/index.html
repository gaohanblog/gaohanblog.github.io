leetCode-460-LFU缓存

发表于 2020-03-02 更新于 2020-03-03 分类于 数据结构与算法 ， leetcode
本文字数： 6.7k 阅读时长 ≈ 6 分钟

题目描述

设计并实现最不经常使用（LFU）缓存的数据结构。它应该支持以下操作：get 和 put。

get(key) - 如果键存在于缓存中，则获取键的值（总是正数），否则返回 -1。
put(key, value) - 如果键不存在，请设置或插入值。当缓存达到其容量时，它应该在插入新项目之前，使最不经常使用的项目无效。在此问题中，当存在平局（即两个或更多个键具有相同使用频率）时，最近最少使用的键将被去除。

示例 1:

LFUCache cache = new LFUCache( 2 /* capacity (缓存容量) */ );

cache.put(1, 1);

cache.put(2, 2);

cache.get(1); // 返回 1

cache.put(3, 3); // 去除 key 2

cache.get(2); // 返回 -1 (未找到key 2)

cache.get(3); // 返回 3

cache.put(4, 4); // 去除 key 1

cache.get(1); // 返回 -1 (未找到 key 1)

cache.get(3); // 返回 3

cache.get(4); // 返回 4

进阶：

你是否可以在 O(1) 时间复杂度内执行两项操作？

解题思路

题目中说明当当缓存达到其容量时，它应该在插入新项目之前，使最不经常使用的项目无效，当存在平局（即两个或更多个键具有相同使用频率）时，最近最少使用的键将被去除。最一开始思路就是用一个双向链表保存所有保存的节点，但是需要维护双向链表一个特性就是，链表中的每个节点的权重(即每个节点的访问次数)是从前到后依次降低的。梳理下思路：

  • 当声明了cache的capaticy为3，put(1, 1)时，先存入map，此时键1的权重为1。为了保存权重声明一个权重的map(weightMap),weightMap的键为添加的键，值为权重，并把这个键追加到双向链表中。
  • put(2, 1) 此时键2的权重也为1，但是题目中说了相同使用频率的键值对时，最近最少使用的键的权重最小。所以为了维护双向链表权重递减的特性，应该添加到键为1的前面。此时的链表的结构为2->1
  • put(3,1) 和put(2,1)一样, 此时的链表的结构为3->2->1
  • get(1),这时1的权重增加1，所以链表的结构为1->3->2
  • put(4,1),这时达到容量，需要先删除一个节点即链表最后一个，此时4的权重为1，此时的链表的结构为1->4->3
    …

但是这样做，复杂度比较大，几乎每进行一次操作都要遍历链表权重维护一次链表(更改节点的权重，改变链表的结构，还需要维护两个map),时间复杂度是O(N)的。在提交的时候，碰到最后两个特别大量的put和get操作的测试时候，直接超时了~

下面看怎么用O(1)的时间实现。

用O(1)时间实现需要利用LinkedHashSet(插入有序),和上面一样也用一个辅助map(weightMap)来保存权重，不同的是weightMap的键保存权重，值为一个LinkedHashSet，还需要记录一个min的全局变量来储存当前最小权重。 下面看下实现的过程：

  • 当声明了cache的capaticy为3，put(1, 1)时，此时weightMap键的权重为1，值为一个LinkedHashSet，此时将put的键作为LinkedHashSet的第一个元素。此时最小权重为1。
  • put(2, 1)，新创建一个LinkedHashSet中添加第二个元素。
  • put(3,1)，新创建一个LinkedHashSet中添加第二个元素。
  • get(1)，再执行get(1)时，键1的权重变为2，这时候，先把原来LinkedHashSet中的1删除，在weightMap中新增加一个键为2(值为新创建LinkedHashSet，并添加1进去)的键值对。
  • put(4,1),这时达到容量，根据min找到最小权重的set，删除set中的第一个元素，并把4添加到权重为1的set中去。

上面这样方法，在weightMap的插入是O(1)的，在容量满了的情况下LinkedHashSet中删除也是O(1)的，新添加也是O(1)的，所以相对于维护双向链表权重的做法是更好的。
