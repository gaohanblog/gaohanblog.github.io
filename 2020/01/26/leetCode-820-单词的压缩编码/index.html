leetCode-820-单词的压缩编码

发表于 2020-01-26 分类于 数据结构与算法 ， leetcode
本文字数： 2.1k 阅读时长 ≈ 2 分钟

题目描述

给定一个单词列表，我们将这个列表编码成一个索引字符串 S 与一个索引列表 A。

例如，如果这个列表是 ["time", "me", "bell"]，我们就可以将其表示为S = "time#bell#" 和 indexes = [0, 2, 5]。

对于每一个索引，我们可以通过从字符串 S 中索引的位置开始读取字符串，直到 “#” 结束，来恢复我们之前的单词列表。

那么成功对给定单词列表进行编码的最小字符串长度是多少呢？

示例：

输入: words = [“time”, “me”, “bell”]

输出: 10

说明: S = “time#bell#” ， indexes = [0, 2, 5] 。

提示：

  • 1 <= words.length <= 2000
  • 1 <= words[i].length <= 7
  • 每个单词都是小写字母 。

解题思路

这个题目的思路就是就是如果有一个单词的后缀是另外一个单词，则可以将这两个单词合并在一起，例如示例中的time和me，第一个单词的后缀me是第一个单词，所有我们可以将它们合并为time#。

解决这个问题最经典的做法就是用字典树，题目要求一个单词的后缀为另一个单词，所以需要将每个单词倒序插入树中（这样就能实现一个单词后缀是另外一个单词的插入）。

例如time倒序是emit依次插入字典树，假如在插入me,发现字典树中已经存在了em的前缀，所以不会在生成新的节点。

遍历结果时，树的每个叶子节点的单词都不为其他单词的后缀，就形成了单词数组的最短索引字符串。
